# RLHF 管道集成可行性评估

> DPO / PPO / GRPO 在 4 个项目中的应用分析 | 2026-06-18

---

## 一、RLHF 三方法简介

| 方法 | 核心思想 | 需要 |
|------|---------|------|
| **DPO** | 直接优化偏好对 (好/坏回答) | 偏好数据集 |
| **PPO** | 强化学习 + 奖励模型 | 奖励模型 + 环境 |
| **GRPO** | 群体相对策略优化 | 群体采样 + 奖励函数 |

---

## 二、四项目匹配度评估

### Kronos Stock — 🟡 低匹配 (30%)

| 维度 | 评估 |
|------|------|
| 模型类型 | LSTM 回归模型 — 非生成式 |
| 偏好信号 | 无天然偏好对 (涨跌没有"好/坏") |
| RLHF 适用性 | ❌ 不适合 DPO/PPO |
| **GRPO 可能** | ✅ 可将 Sharpe Ratio 作为奖励函数，直接优化交易策略 |

**方案**: 用 GRPO 替代 MSE 损失，直接最大化 Sharpe Ratio。参考 FareedKhan 的 GRPO 实现。

```
当前: LSTM → MSE(预测收益, 实际收益) → RankIC +0.205
GRPO: LSTM → 采样多条轨迹 → 计算每条Sharpe → 优化高Sharpe方向
```

### 医疗文本 — 🟢 高匹配 (90%) 🔥 最佳

| 维度 | 评估 |
|------|------|
| 模型类型 | Qwen3-0.6B 生成式 LLM |
| 偏好信号 | ✅ 已有 449 QA对, 可构造偏好 |
| DPO 适用性 | ✅ 完美匹配 |
| 现有基础 | ✅ 已完成指令微调 (ChatML) |

**方案**: 在指令微调后添加 DPO 阶段，用偏好数据对齐输出质量。

### 法规文本 — 🟡 中匹配 (60%)

| 维度 | 评估 |
|------|------|
| 模型类型 | Qwen3-0.6B 生成式 LLM |
| 偏好信号 | ⚠️ 仅有续写数据, 无偏好对 |
| DPO 适用性 | ✅ 可行但需先构造偏好数据 |

**方案**: 需先生成偏好数据 (好/坏法规文本)，再进行 DPO。

### 控费 ML — 🔴 不匹配 (10%)

| 维度 | 评估 |
|------|------|
| 模型类型 | LightGBM 树模型 |
| RLHF 适用性 | ❌ 完全不适用 |
| 原因 | 树模型无序列生成, 无 token 级别优化 |

## 四、推荐方案: 医疗项目 DPO

### 为什么选医疗

1. **已有指令微调基础**: Qwen3-0.6B + LoRA + ChatML
2. **已有 QA 数据**: 449 对, 可构造偏好
3. **模型兼容**: 生成式 LLM, DPO 天然适用
4. **FareedKhan 参考**: 其 DPO 实现可直接参考

### 实施计划

```python
# Step 1: 构造偏好数据
# 对每个医学问题, 生成2个回答, 用 LLM-as-Judge 打分

# Step 2: DPO 训练 (参考 FareedKhan train_dpo.py)
# 损失: L_DPO = -log σ(β(log π_θ(y_w|x) - log π_ref(y_w|x))
#                    - β(log π_θ(y_l|x) - log π_ref(y_l|x)))

# Step 3: 评估
# 对比 DPO前/后 的生成质量
```

### 预期效果

| 指标 | DPO前 | DPO后 |
|------|-------|-------|
| 回答完整性 | 基线 | +10-20% |
| 医学术语准确性 | 基线 | +5-10% |
| 格式规范性 | 基线 | +15-25% |
