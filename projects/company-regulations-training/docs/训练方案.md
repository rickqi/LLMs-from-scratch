### 数据清洗规则

```python
# 1. 去除 HTML 标签
text = re.sub(r"<[^>]+>", "", text)

# 2. 去除图片标记 (PDF 提取残留)
text = re.sub(r"!\[.*?\]\(.*?\)", "", text)

# 3. 去除 OCR 页码
text = re.sub(r"page=\d+[^)]*\)", "", text)

# 4. 去除 Markdown 标题符号 (保留文本)
text = re.sub(r"^#{1,6}\s*", "", text, flags=re.MULTILINE)

# 5. 去除水印/引用行
text = re.sub(r"^>\s*.*$", "", text, flags=re.MULTILINE)

# 6. 合并多余空行
text = re.sub(r"\n{4,}", "\n\n", text)
```

## 四、训练配置

### 超参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 基座模型 | Qwen/Qwen3-0.6B | 通过 hf-mirror.com 下载 |
| LoRA rank | 8 | 低秩维度 |
| LoRA alpha | 16 | 缩放因子 (= 2×rank) |
| LoRA dropout | 0.05 | 正则化 |
| Target modules | q/k/v/o_proj | 注意力层全部 |
| Batch size | 4 | 物理 batch |
| Gradient accumulation | 4 | 有效 batch = 16 |
| Max sequence length | 512 | 上下文窗口 |
| Epochs | 5 | 训练轮数 |
| Learning rate | 2e-4 | 峰值学习率 |
| Warmup | 5% total steps | 线性预热 |
| Scheduler | Linear decay | 线性衰减 |
| Gradient clipping | max_norm=1.0 | 防梯度爆炸 |
| Weight decay | (AdamW 默认) | L2 正则化 |
| Precision | bfloat16 (GPU) | 混合精度 |

## 五、训练流程

### Pipeline

```
Step 1: 数据准备 (data_prep.py)
  /home/docs/raw/公司规章制度/*.md
    ↓ 清洗 (去 HTML/OCR/图片标记)
    ↓ 分割 (95% train / 5% val)
    ↓ 格式化 (===SEP=== 分隔)
  data/train.txt (53.2 MB)
  data/val.txt (1.8 MB)

Step 2: 训练 (train_qwen_lora.py)
  Qwen3-0.6B + LoRA (r=8)
    ↓ AdamW(lr=2e-4) + Warmup + Cosine Decay
    ↓ 5 Epochs, batch=4, grad_accum=4
    ↓ 每 10 步评估 + 保存最佳模型
    ↓ 每 epoch 保存断点 + 生成样本
  output/best_model/ (LoRA 权重)
  output/training_log.json (训练曲线)

Step 3: 推理 (generate.py)
  加载 Qwen3-0.6B + LoRA
    ↓ 输入提示词 (制度名称/适用范围/管理职责...)
    ↓ 生成公司制度文本
  评估输出质量
```

### 断点续训

```bash
# 中断后重新运行, 自动从上一个 epoch 恢复
python train_qwen_lora.py --data_dir ./data --output_dir ./output
# → "从 epoch 3, step 150 恢复 (best_val=1.2345)"
```

| 0 | ~3.12 | ~2.91 | 初始 |

| 500 | 2.30 | 2.05 | 持续学习 |
| 1000 | 2.18 | 1.95 | 减速 |
| 2100 | 2.10 | **1.90** | 当前 best_model |

### LoRA 权重影响分析

通过 `merge_and_unload()` 对比基座与微调后权重：

| 指标 | 值 |
|------|-----|
| 基座模型总参数 | 596.0M (Qwen3-0.6B) |
| LoRA 训练参数 | 2.3M (0.38%) |
| 实际影响基座参数 | 176.2M (**29.6%**) |
| 放大倍数 | 76.8x (每 LoRA 参数影响 ~77 个基座参数) |
| 平均变化幅度 | 0.0008 |
| 最大变化幅度 | 0.030 (o_proj) |

按模块分布：

| 模块 | 层数 | 修改率 | 说明 |
|------|------|--------|------|
| q_proj | 28 | 100% | 全部修改 |
| k_proj | 28 | 100% | 全部修改 |
| v_proj | 28 | 100% | 全部修改 |
| o_proj | 28 | 100% | 全部修改 |
| FFN/embed | — | 0% | 完全冻结 |

关键结论：LoRA 只训练 0.38% 参数，通过低秩矩阵乘法影响了 29.6% 的基座参数，但变化幅度极小 (平均 0.0008)，在保持预训练知识的同时实现了领域适应。

### 脚本优化记录

| 改动 | 原因 |
|------|------|
| `evaluate(max_batches=30)` | 原全量 eval 2273 样本需 19 分钟，限制到 30 batch (~1 分钟) |
| eval 频率 10→20 steps | 减少评估开销 |
| 新增每 50 batch 进度日志 | 实时监控训练进度 |
| `PYTHONUNBUFFERED=1` | 解决 tee 管道缓冲导致日志不输出 |
