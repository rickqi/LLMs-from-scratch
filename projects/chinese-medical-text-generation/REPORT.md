# Qwen3 + LoRA 中文医学文本生成 — 训练分析报告

> 生成日期: 2026-06-19 | 模型: 0.6B / 1.7B | Phase 1+2 全流程完成

## 一、项目概述

使用 Qwen3-0.6B (600M 参数) + LoRA (rank=8) 对 374 篇中文医学文档进行持续预训练微调，使模型学习中国医学诊疗指南的语言模式。

**数据来源**: `/home/raw/medica/` (107MB, 4 大类)

| 分类 | 篇数 | 内容 |
|------|------|------|
| L1_卫健委官方规范 | 158 | WHO指南/CDC传染病/肿瘤诊疗指南/安宁疗护 |
| L2_卫生行业标准_WS | 59 | 临床检验/医院感染/基层医疗/老年健康/放射影像 |
| L3_中华医学会临床诊疗指南丛书 | 146 | CACA癌种指南/CMA专家共识/36分册临床诊疗指南 |
| L4_中华医学会临床技术操作规范 | 49 | 内科/外科/妇产/儿科/眼科/影像/核医学等 |

## 二、实验历程

### 2.1 迭代概览

| 版本 | 数据 | 样本数 | 配置 | val_loss | 耗时 | 评估 |
|------|------|--------|------|---------|------|------|
| V1 合成 | 合成模板 200篇 | 475 | batch=4, ep=5, rank=8 | 0.5506 | 1.7min | 模板填充，后期重复 |
| V2 合成 | 合成模板 500篇 | 1,172 | batch=8, ep=10, rank=8 | 0.3304 | 4.3min | 更多模板，仍过拟合 |
| V3 合成 | 合成模板 1000篇 | 2,700 | batch=8, ep=15, rank=8 | 0.2653 | 10.1min | 最低 loss，但严重过拟合 |
| 肿瘤-10 | 真实CACA 10篇 | 1,172 | batch=8, ep=3, rank=8 | 3.1475 | 25min | 首次真实数据，质量好 |
| 肿瘤-51 | 真实CACA+CMA 51篇 | 5,400 | batch=4, ep=6, rank=16 | 2.6180 | ~3h | rank=16 过拟合，epoch 6 退化 |
| **全量-374** | **全部 374 篇** | **29,650** | **batch=4, ep=5, rank=8** | **2.4033** | **11.0h** | **✅ 最终模型** |

### 2.2 关键决策节点

```
合成数据探索 (1.7-10min)
    │
    ├─ 发现: 模板数据 val_loss 极低 (0.27) 但生成重复
    │         → loss 数字不可跨数据集对比
    │
    ▼
真实数据验证 (25min-3h)
    │
    ├─ 发现: 真实数据 loss 高 (3.15) 但生成质量远超合成
    │         → 数据多样性 > loss 数值
    ├─ 发现: rank=16 在 51 篇时 epoch 6 已过拟合
    │         → 秩需匹配数据量
    ├─ 发现: truncation 丢弃 99% 数据
    │         → 修复滑动窗口，数据利用率 100%
    │
    ▼
全量训练 (11.0h)
    │
    ├─ 374 篇全科室覆盖，29,650 滑动窗口样本
    ├─ rank=8, 5 epochs → val=2.4033
    ├─ 预判收敛值 ≈ 2.40 → 实际 2.4033 ✅
    └─ train/val gap ≈ 0.11，无过拟合
```

## 四、生成质量评估

### 4.1 评估 Prompt 生成结果

**临床表现**：
> 起病缓慢，一般无发热症状，皮疹出现缓慢。皮疹初期为淡红色斑块或丘疹，随后渐红、硬结或结节...血清学检查：抗红细胞抗体阳性。

**诊断依据**：
> 反复发作的阵发性心绞痛。心电图和冠状动脉造影显示冠状动脉内壁有斑块或狭窄...胸片、血常规、心肌酶谱、心导管检查。

**治疗方案**：
> 静脉注射泼尼松50~100mg/(kg·d)，分2~3次口服，疗程6~12周...环孢素A、硫唑嘌呤。

**预后判断**：
> 中位生存时间6.18个月...放疗方案为同步放化疗（ICCI-ICCI），术后辅助放疗。

**鉴别诊断**：
> 多发性骨髓瘤、多发性骨髓瘤伴淋巴结和肝脾肿大...骨髓穿刺病理学检查以明确诊断。

### 4.2 多科室覆盖验证

| 科室 | 生成内容 |
|------|---------|
| 心血管 | 阵发性心绞痛、西他卡因/美托洛尔、冠状动脉痉挛 |
| 肿瘤 | 顺铂+依托泊苷、多西他赛+培美曲塞、肝癌化疗方案 |
| 呼吸/结核 | 异烟肼、利福平、结核性脑膜炎、胸膜转移 |
| 皮肤 | 淡红色斑块/丘疹、维A酸霜剂、糖皮质激素类药物 |
| 血液 | 多发性骨髓瘤、血红蛋白95~100g/L、骨髓穿刺 |
| 骨科 | 原发性骨肉瘤、骨髓转移性恶性肿瘤、骨膜反应 |
| 内分泌 | 甲状腺癌切除术、乳头状/滤泡状/髓样亚型 |

### 4.3 质量评分

| 维度 | 评分 | 说明 |
|------|------|------|
| 中文流畅度 | ⭐⭐⭐⭐⭐ | 通顺自然，无拼接痕迹 |
| 医学术语准确性 | ⭐⭐⭐⭐ | 真实药物名/病理术语，偶有幻觉 |
| 结构规范性 | ⭐⭐⭐⭐⭐ | 临床表现→诊断→治疗→预后，格式正确 |
| 科室多样性 | ⭐⭐⭐⭐⭐ | 覆盖肿瘤/心血管/呼吸/感染/皮肤/骨科 |
| 幻觉程度 | ⭐⭐⭐ | 部分药物剂量/方案为编造，需事实核查 |

### 4.4 已知局限

1. **OCR 残留**: 部分输出含 `$\textcircled{1}$` (LaTeX 编码的 ①)
2. **剂量幻觉**: 药物剂量和联合方案可能不准确
3. **长输出重复**: 超过 500 token 后偶有循环
4. **中英混杂**: 偶见英文术语 (如 CDDP/PAS)，来自原始 CACA 英文指南

## 六、关键经验教训

### 6.1 数据相关

| 教训 | 详情 |
|------|------|
| **truncation bug** | 原代码 `truncation=True` 对长文档只取前 512 token，丢弃 99% 内容。修复为滑动窗口 (stride=512) 后数据利用率 100% |
| **Loss ≠ 质量** | 合成数据 val=0.27 远低于真实数据 val=2.40，但生成质量完全相反。Loss 依赖数据分布 |
| **全量 > 精选** | 374 篇全科室覆盖比 51 篇肿瘤专科的泛化能力强得多 |
| **中英混合输入** | CACA 英文指南混入中文训练引入英文片段，建议预过滤 |

### 6.2 模型相关

| 教训 | 详情 |
|------|------|
| **rank=8 优于 rank=16** | 374 篇数据下 rank=8 不出现过拟合，rank=16 在 epoch 6 已劣化 |
| **epoch 不是越多越好** | Epoch 5 边际贡献仅 0.01，过拟合风险 > 收益 |
| **train/val gap 监控** | gap ≈ 0.11 为健康值，若 gap > 0.3 提示过拟合 |
| **Qwen3 是最优解** | ≤1B 范围内除 Qwen 外无任何模型有中文能力 |

### 6.3 工程相关

| 教训 | 详情 |
|------|------|
| **断点续训是生命线** | 11h 训练不可能一次完成，checkpoint 支持中断恢复 |
| **batch=8 OOM** | RTX 5080 16GB 无法跑 batch=8 + max_len=512，降为 4 |
| **后台运行** | `nohup` + 日志文件是长时间训练的标准方式 |
| **评估频率** | eval_freq=10 (40 batch) 在 7K batch/epoch 下产生 ~185 日志点 |

## 九、对比实验：LoRA 模型 vs AI 知识检索

### 9.1 实验设计

用同一个问题测试两个系统，对比输出差异：

```
问题: 甲状腺切除手术对身体有什么影响？晕倒风险如何？
```

**系统 A (AI 知识检索)**：AI 助手从自身预训练数据中检索医学知识，结构化组织回答。

**系统 B (LoRA 模型)**：Qwen3-0.6B + LoRA adapter，通过 `generate.py` 加载，多次尝试不同 prompt 格式。

### 9.2 系统 A 输出 (AI 知识检索)

> **短期影响**：颈部疼痛/吞咽不适(90%+), 声音嘶哑(5-15%), 低钙血症(10-30%), 颈部血肿(<2%), 伤口感染(<1%)
>
> **长期影响**：甲状腺功能减退(全切100%→终身左甲状腺素), 永久性声音嘶哑(<2%), 永久性低钙(<5%)
>
> **晕倒风险**：低钙血症→喉痉挛, 体位性低血压, 颈部血肿→气管+颈动脉窦压迫(最危急🔴), 甲状腺危象(<1%), 贫血/低血容量
>
> **预防**：血钙监测, 半卧位, 气管切开包, 下床三步法, 终身优甲乐

### 9.3 系统 B 输出 (LoRA 模型)

**Prompt 1**: `甲状腺切除手术对身体有什么影响？`

> 1. 突然停止或中断甲状腺功能可导致代谢紊乱，如心律失常、电解质紊乱等。
> 2. 术后应密切观察病人有无出现异位甲状腺功能亢进的症状，如手抖、多汗、易怒、焦虑...
> 3. 术后病人常有心理变化，因此心理治疗和心理支持非常重要。
> 4. 术后早期出现的甲状腺功能亢进症状，需要及时处理。对甲状腺肿大而有明显压迫症状者，**应行全喉切除术**。若颈部淋巴结转移，则应考虑进行**全喉切除及颈部淋巴结清扫术**。
> 5. 术后病人应注意保暖，避免受凉，防止感冒。术后1周内禁止剧烈运动。
> 6. 颈部淋巴结转移者，应给予充分的镇痛，以缓解疼痛。
> 7. 应注意预防感染...避免辛辣刺激食物...

**Prompt 2**: `甲状腺切除术后并发症及处理措施：`

> 1) 恶性肿瘤切除后，应定期复查。2) 术后早期易复发，可酌情延长随访时间。3) 需要根治性切除者，应密切观察有无复发。
> **（三）甲状腺癌的诊断**
> 甲状腺癌的确诊主要依靠病理检查，包括活检、切片、组织化学和免疫组化检查等。
> **乳头状甲状腺癌** — 占本病的90%～95%，其中70%～80%发生于5岁以下儿童...囊内出血可使囊壁变薄、液面变高...

**Prompt 3**: `【术后管理】甲状腺切除术后晕倒的风险评估与预防`

> 术后2周内应避免剧烈活动，术后1个月内不宜从事体力劳动。
> 术后疼痛的处理：术后疼痛主要表现为局部疼痛，可采用止痛药物、镇痛泵或神经阻滞等方法缓解。
> 术后护理要点：①术后早期(1~2天)卧床休息，轻度活动，注意观察患者是否出现心前区不适症状；②术后第3天开始逐步进行下肢功能锻炼；③术后2周内应避免剧烈活动；④术后2周内应避免用力过猛；⑤术后3周内应加强生活护理。
> 术后切口感染：切口部位有渗血、脓性分泌物，伤口周围红肿明显...

### 9.4 差异对比

| 维度 | AI 知识检索 (系统A) | LoRA 模型 (系统B) |
|------|-------------------|-------------------|
| **回答方式** | 直接回答问题，给出结构化的"是什么" | 续写文本，生成指南风格的"怎么做" |
| **准确性** | ✅ 高（完全切除→甲减，非全喉切除） | ⚠️ 存在事实错误（如"全喉切除术"用于甲状腺） |
| **信息组织** | 表格/分级列表，短期→长期→风险→预防 | 流水账式编号列表，无明确结构 |
| **晕倒风险** | 明确列出 5 种机制+严重度分级 | 未直接提及晕倒，仅提到"观察心前区不适" |
| **数据支撑** | 发生率数据 + 风险等级 + 机制解释 | 操作步骤 + 注意事项，缺乏量化数据 |
| **OCR 干扰** | 无（纯文本） | 有（$\textcircled{1}$, $\mathrm{T}_{3}$） |
| **话题聚焦** | 严格围绕问题 | 易漂移（甲状腺切除→甲状腺癌诊断→乳头状癌病理） |
| **格式** | Markdown 表格/标题/加粗 | 编号列表，指南结构 |
| **可靠性** | 可作参考，需核实 | 仅作生成示例，不可直接采信 |
| **用户友好度** | 一目了然 | 需要筛选和判断有效信息 |

### 9.5 根因分析

LoRA 模型表现出"续写"而非"问答"行为，根因在于训练范式：

```
训练数据格式:
  原始 .md 文档 → 滑动窗口切片 → 512 token 段落
  例: "甲状腺切除术后并发症及处理措施：1) 恶性肿瘤切除后..."

模型学到的:
  "当看到 '甲状腺切除...' 开头 → 生成后续医学段落"
  而非: "当被问到 '甲状腺切除有什么影响？' → 给出结构化回答"
```

**这解释了所有差异**：

| LoRA 模型行为 | 根因 |
|--------------|------|
| 列出编号操作步骤 | 训练数据是医学指南，天然就是编号列表 |
| 漂移到甲状腺癌诊断 | 训练数据中甲状腺+肿瘤=诊断章节连续出现 |
| 出现"全喉切除术" | 训练数据中颈部手术混合了喉+甲状腺内容 |
| 不区分风险等级 | 原始指南未按风险等级组织 |
| 无发生率数据 | 原始指南多为操作规范，非统计报告 |

### 9.6 修复路径：指令微调

指令微调（阶段 2）可改变模型行为，使其从"续写"变为"问答"：

```
训练数据格式 (ChatML):
  <|im_start|>user
  甲状腺切除手术对身体有什么影响？
  <|im_end|>
  <|im_start|>assistant
  甲状腺切除手术的影响分为短期和长期...
  短期：颈部疼痛(90%+)、声音嘶哑(5-15%)...
  长期：全切患者终身服用左甲状腺素...
  <|im_end|>
```

| 阶段 | 能力 | 系统 B (当前) | 指令微调后 |
|------|------|-------------|-----------|
| 文本续写 | 医学指南风格续写 | ✅ | ✅ |

### 13.2 阻塞根因

所有 Qwen3/Qwen3.5 新型号 (非 0.6B) 均通过 `cas-bridge.xethub.hf.co` 分发，该 CDN 从当前环境访问极慢（~100KB/s），导致：
- 0.8B (~1.6GB) → 预计 4h+ 下载
- 1.7B (~3.4GB) → 预计 9h+ 下载
- 2B (~4GB) → 预计 11h+ 下载

**而 Qwen3-0.6B 已缓存在 hf-mirror 常规 CDN，秒级加载。**

### 13.3 缓解方案

| 方案 | 操作 | 效果 |
|------|------|------|
| 预下载到本地 | `huggingface-cli download Qwen/Qwen3.5-2B-Base --local-dir ./models/2b` | 一次性下载，后续秒加载 |
| 换代理 | `export HF_ENDPOINT=https://hf-mirror.com` (已设置) | 无效，mirror 也走 cas-bridge |
| 用已缓存模型 | `Qwen3-0.6B-Instruct` (已缓存) | 秒启动，指令能力对比 |

### 13.4 实际对比结果 (10篇 CACA 肿瘤指南)

| 指标 | Qwen3-0.6B | Qwen3.5-2B |
|------|-----------|-----------|
| 训练参数 | 2,293,760 (0.30%) | 737,280 (0.04%) |
| val_loss (epoch1) | 2.88→2.51 | 2.71→2.51 |
| val_loss (最终) | 3.148 | **2.457** |
| 耗时 | 25min (epoch1) | **10h** (3 epochs) |
| 速度 | 1x | **24x 慢** |

| LoRA 权重 | 8.8MB | 2.9MB |

**生成质量对比**:

| 速度"提升" | DPO 输出更短 → 生成更快，非推理效率提升 | ℹ️ |
| Q2 反向建议 | Inst-V2 独立问题，DPO 修复了此具体 case | 🔴 |

**核心结论**: DPO 偏好对齐未能稳定改善质量。586 对 DPO 数据的 `chosen/rejected` 配对质量控制不足——长度差异、格式差异、内容准确性混杂在同一对中，导致模型学到了错误的偏好信号。

## 十八、DPO 改进方案

### 18.1 问题三层结构

```
表层: Q3 输出坍缩为 29 tokens, Q8 缩为 115 tokens
  ↓
中层: DPO 偏好信号被长度污染 — chosen/rejected 不仅区分质量，也编码了长度偏好
  ↓
深层: 586 对数据中正负样本的差异维度不纯
      (长度差 ±40%, 格式差异, 内容准确性混杂在同一对中)
```

### 18.2 改进方案矩阵

| 优先级 | 方案 | 具体操作 | 预期效果 |
|--------|------|---------|---------|
| 🔴 P0 | **长度归一化配对** | 确保 chosen/rejected 长度差 < 20%，必要时截断 chosen 或补全 rejected | 消除长度偏好 → 解坍塌 |
| 🔴 P0 | **人工审核清洗** | 抽检 50 对样本，确保 chosen 在准确性/完整性上明确优于 rejected | 提升信号纯度 |
| 🟠 P1 | **分层配对策略** | 按问题类型分组（知识类/对比类/列举类），同类内 DPO | 避免类型混淆 |
| 🟠 P1 | **降低 beta** | 0.1 → 0.05，减弱偏好信号强度 | 防过拟合 |
| 🟡 P2 | **减少 epochs** | 3 → 1 epoch，DPO 收敛极快 | 减少偏好过拟合 |
| 🟡 P2 | **DPO + SFT 混合训练** | 80% DPO + 20% SFT (原始指令数据 607 QA) | 保持基础能力，防坍塌 |
| 🟢 P3 | **1.7B DPO** | 在 1.7B Inst-V1 上重做 DPO，更大模型抗坍塌能力更强 | 质量跃升 |
| 🟢 P3 | **自动生成 better DPO 数据** | 用 1.7B model 生成 chosen，0.6B 生成 rejected，人工筛选 | 提升数据质量 |

### 18.3 推荐执行路径

```
Step 1 (P0) — 数据修复 (~1h)
  ├── 分析 586 对中 chosen/rejected 的长度分布
  ├── 过滤长度差 > 30% 的 pair
  ├── 人工审核 50 对，标记明显不合理的配对
  └── 目标: 保留 300+ 高质量 pair

Step 2 (P1) — 重新训练 (~30min)
  ├── python train_dpo.py --epochs 1 --beta 0.05 --batch_size 2
  └── 监控: 确保 val 不出现 loss 暴跌 (坍塌前兆)

Step 3 (P2) — 验证 (~15min)
  ├── 9 题评测 → 确认无坍塌 (所有题 > 100 tokens)
  └── 人工评估 3 题 → 确认质量提升

Step 4 (P3, 可选) — 规模化
  └── 1.7B DPO or 自动生成 better pairs
```
