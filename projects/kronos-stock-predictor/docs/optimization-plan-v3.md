## 二、优化方案 (按优先级)

### 🔴 P0: 全量数据 + 长 epochs

```
目标: 最大化训练数据, 收敛模型
数据: 全部 183只, 2018-2025 (8年)
测试: 2026 (保留半年盲测)
epochs: 50 (静态) + Walk-forward 30/季
预期: 静态 Sharpe > 2.5, Walk-forward RankIC > 0
```

### 🔴 P1: 周度 Walk-forward

```
目标: 增加观测点 (6→78), 提升统计效力
数据: 同上
频率: 每周滚动预测, 而非每季
预期: 78 个数据点, p-value 大幅降低
```

### 🟡 P2: 分行业波动率扫描

```
目标: 找到波动率信号最强的行业板块
范围: 半导体/医药/消费/新能源/金融
方法: 每个板块独立训练 LSTM → 比较 Walk-forward RankIC
预期: 识别 2-3 个正 RankIC 板块
```

### 🟢 P3: Transformer 架构

```
目标: 更大模型容量 → 更丰富的 token 表征
模型: Kronos BSQ Tokenizer + Transformer Predictor
数据: 183只 全量
预期: 在充分数据下可能超越 LSTM
```

## 三、执行计划

```
Phase 1 (立即):
  □ 数据: 183只, train 2018-2025, test 2026
  □ 静态训练: 50 epochs, 记录 Sharpe + RankIC
  □ Walk-forward 30ep/季: 6个季度
  □ 对比: 50ep vs 30ep vs 20ep

Phase 2 (本周):
  □ 周度 Walk-forward: 每周滚动, 78 数据点
  □ 分行业扫描: 5 个板块独立评估

Phase 3 (本月):
  □ Kronos Transformer 训练 + 对比
  □ 蒙特卡洛重新检验
```
