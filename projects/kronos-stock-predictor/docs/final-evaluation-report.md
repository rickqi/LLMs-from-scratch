## 二、瓶颈分析

### 2.1 核心瓶颈：模型架构 × 数据规模 不匹配

```
数据量: 68 stocks, ~60k samples, 6 features (OHLCV)
模型: decoder-only Transformer + BSQ tokenizer

问题: 自回归 Transformer 在 6 维 OHLCV 的离散 token 上，
      60k 样本不足以学习稳定的跨品种预测信号。
```

### 2.2 已验证的无效路径

| 尝试 | 失败原因 |
|------|---------|
| 扩大模型 (small) | 参数↑但数据不变 → 过拟合 |
| 扩大品种 (68→96) | 跨行业噪声稀释板块特有模式 |
| 技术指标 (19feat) | 指标噪声 > 信号，tokenizer容量浪费 |
| 收益率建模 | 收益率比价格更随机，不可预测 |
| Kronos-small 预训练 | 全球化预训练≠中国半导体板块 |

### 2.3 根本原因

1. **数据规模天花板**：60k 样本对于 Transformer 来说偏少
2. **特征维度天花板**：仅 OHLCV 6 维，金融信号天然低信噪比
3. **BSQ token 离散化损失**：连续→离散的量化误差淹没微弱信号
4. **单板块限制**：半导体高度同质化，缺少负样本多样性

## 三、下一步计划

### 🔴 不建议（已验证无效）

| 行动 | 证据 |
|------|------|

| 添加更多特征到 tokenizer | 实验#6 恶化 |

### 🟢 建议尝试

| 优先级 | 方向 | 核心思路 | 可行性 |
|--------|------|---------|--------|

| **P1** | 简化模型 | 用更简单的 LSTM 替代 BSQ+Transformer | 高 |
