## 一、这本书教什么

```
第1章(理论) → 第2章(分词+嵌入) → 第3章(注意力) → 第4章(GPT模型)
    → 第5章(预训练) → 第6章(分类微调) → 第7章(指令微调)
```

### 📇 第 1 章：理解大型语言模型

> 第 1 章无代码，为纯理论章节。作者推荐阅读方法详见 `ch01/reading-recommendations.md`。配套视频：[环境配置](https://www.youtube.com/watch?v=yAcWnfsZhzo)、[LLM 生命周期](https://www.youtube.com/watch?v=kPGTx4wcm_w)。

<details>
<summary><b>Q1: LLM 的核心任务是什么？</b></summary>

**A:** 自回归语言建模——给定前文 token 序列，预测下一个 token 的概率分布。

$$P(w_t | w_1, w_2, ..., w_{t-1})$$

本质是一个超大规模的分类器，每次从整个词汇表中选下一个最可能的词。

</details>

<details>
<summary><b>Q2: GPT 类模型的三个训练阶段是什么？</b></summary>

**A:**
1. **预训练（Pretraining）**：在大型无标签文本语料上训练，学习通用语言知识
2. **监督微调（SFT / Finetuning）**：在下游任务标注数据上微调
3. **对齐（Alignment）**：通过 RLHF 或 DPO 让模型输出符合人类偏好

</details>

<details>
<summary><b>Q3: 124M GPT-2 的关键配置参数有哪些？</b></summary>

**A:**
```
vocab_size     = 50257    # 词汇表大小
context_length = 1024     # 最大上下文长度
emb_dim        = 768      # 嵌入维度
n_heads        = 12       # 注意力头数
n_layers       = 12       # Transformer 层数
drop_rate      = 0.1      # Dropout 比例
qkv_bias       = False    # QKV 投影是否使用 bias
```

</details>

### 📇 第 2 章：处理文本数据

<details>
<summary><b>Q4: BPE（Byte Pair Encoding）分词的基本原理是什么？</b></summary>

**A:**
1. 将文本拆分为字节级（byte-level）基础字符
2. 统计所有相邻 token 对的出现频率
3. 合并最高频的 token 对，形成新 token
4. 重复步骤 2-3 直到达到目标词汇量大小

**优势**：能处理未见过的词（拆分为子词），是 GPT-2/3/4 使用的分词算法。

</details>

<details>
<summary><b>Q5: 嵌入层（Embedding Layer）做了什么？</b></summary>

**A:** 将离散的 token ID（整数）映射为可学习的连续稠密向量。本质是一个查找表：

```python
# vocab_size=50257, emb_dim=768
tok_emb = nn.Embedding(50257, 768)
# token_id [5, 10, 200] → 三个 768 维向量
```

> 嵌入层等效于：对 one-hot 向量做矩阵乘法（`one_hot @ weight_matrix`）。

</details>

<details>
<summary><b>Q6: 滑动窗口 DataLoader 是如何创建输入-目标对的？</b></summary>

**A:**
```
文本 token 序列: [t1, t2, t3, t4, t5, t6, t7, t8, ...]

max_length=4, stride=2:
  input:  [t1, t2, t3, t4]  target: [t2, t3, t4, t5]
  input:  [t3, t4, t5, t6]  target: [t4, t5, t6, t7]
  input:  [t5, t6, t7, t8]  target: [t6, t7, t8, t9]
```

目标就是输入向右偏移一位——因为 LLM 的任务是预测下一个词。

</details>

### 📇 第 3 章：注意力机制

<details>
<summary><b>Q7: 自注意力的计算公式是什么？写出计算步骤。</b></summary>

**A:**
$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

**步骤**：
1. 从输入 x 投影出 Q、K、V：`Q=x·Wq, K=x·Wk, V=x·Wv`
2. 计算注意力分数：`scores = Q @ K^T`
3. 缩放：`scores / sqrt(d_k)`——防止点积过大导致 softmax 梯度消失
4. 应用因果掩码（causal mask）：将未来位置设为 `-inf`
5. Softmax 归一化：`attn_weights = softmax(scores)`
6. 加权求和：`output = attn_weights @ V`

</details>

<details>
<summary><b>Q8: 因果掩码（Causal Mask）是什么？为什么需要它？</b></summary>

**A:** 一个上三角矩阵，将未来位置设为 `-inf`（softmax 后变为 0）。

```python
# 原始掩码矩阵 (4×4)
[[0,  -inf, -inf, -inf],
 [0,    0,  -inf, -inf],
 [0,    0,    0,  -inf],
 [0,    0,    0,    0 ]]
```

**原因**：LLM 是自回归模型，生成 token[t] 时不能"偷看" token[t+1] 及之后的内容，否则等于作弊。

</details>

<details>
<summary><b>Q9: 多头注意力的张量形状变化是怎样的？</b></summary>

**A:**
```
输入: x                         → (batch, num_tokens, d_in)    例: (2, 6, 768)

Q/K/V 投影:                     → (2, 6, d_out)                 例: (2, 6, 768)

拆分为多头: view+transpose      → (2, n_heads, 6, head_dim)    例: (2, 12, 6, 64)

注意力计算: attn_weights @ V    → (2, n_heads, 6, head_dim)    例: (2, 12, 6, 64)

合并多头: transpose+view        → (2, 6, d_out)                 例: (2, 6, 768)

输出投影: out_proj              → (2, 6, d_out)                 例: (2, 6, 768)
```

</details>

<details>
<summary><b>Q10: 为什么注意力分数要除以 sqrt(d_k)？</b></summary>

**A:** 当 `d_k`（每个头的维度）较大时，点积 `Q·K^T` 的值方差会增大。经过 softmax 后，大的值会趋近于 1，小的趋近于 0——导致梯度几乎为零（softmax 饱和）。

除以 `sqrt(d_k)` 将方差控制为 1，保持 softmax 输出"软"一些，梯度更稳定。

</details>

### 📇 第 4 章：GPT 模型

<details>
<summary><b>Q11: GPTModel 由哪些组件组成？画出结构。</b></summary>

**A:**
```
GPTModel:
├── tok_emb    (nn.Embedding)       词元嵌入
├── pos_emb    (nn.Embedding)       位置嵌入
├── drop_emb   (nn.Dropout)         嵌入 Dropout
├── trf_blocks (nn.Sequential)     12 个 TransformerBlock
│   └── TransformerBlock:
│       ├── norm1       (LayerNorm)
│       ├── att         (MultiHeadAttention)
│       ├── drop_shortcut (Dropout)
│       ├── norm2       (LayerNorm)
│       ├── ff          (FeedForward)
│       └── drop_shortcut (Dropout)
├── final_norm (LayerNorm)          最终归一化
└── out_head   (nn.Linear)          输出投影到词汇表
```

每个 TransformerBlock 内部使用 **Pre-LayerNorm + 残差连接**。

</details>

<details>
<summary><b>Q12: Pre-LayerNorm 和 Post-LayerNorm 有什么区别？本书用哪种？</b></summary>

**A:**
```
Post-LN (原版 Transformer):    Pre-LN (GPT-2/本书):
x → att → add → norm → ff → add → norm    x → norm → att → add → norm → ff → add
```

**Pre-LN 优势**：训练更稳定，不需要 warmup。本书（GPT-2）使用 Pre-LN。

</details>

<details>
<summary><b>Q13: GELU 和 ReLU 激活函数的区别？为什么 GPT 用 GELU？</b></summary>

**A:**

| 函数 | 公式 | 特点 |
|------|------|------|
| ReLU | `max(0, x)` | 简单，但 x<0 时梯度为零 |
| GELU | `x · Φ(x)` | 处处可导，更平滑，实际效果更好 |

GELU（Gaussian Error Linear Unit）在 NLP 任务中表现优于 ReLU，是 GPT/BERT 标配。

</details>

<details>
<summary><b>Q14: 为什么 GPTModel 的输出头 out_head 不使用 bias？</b></summary>

**A:** `out_head` 的权重与词元嵌入 `tok_emb` 共享（权重绑定/weight tying）。不带 bias 使得权重矩阵可以直接复用，减少参数量（124M → 约 107M），且不损失性能。

</details>

### 📇 第 5 章：预训练

<details>
<summary><b>Q15: 语言模型预训练的损失函数是什么？</b></summary>

**A:** **交叉熵损失（Cross-Entropy Loss）**。

```python
loss = F.cross_entropy(
    logits.flatten(0, 1),    # (batch*seq_len, vocab_size)
    targets.flatten()        # (batch*seq_len,)
)
```

对每个位置，模型预测下一个 token 的概率分布，与真实 token 比较计算损失。

</details>

<details>
<summary><b>Q16: 文本生成（贪婪解码）的算法步骤？</b></summary>

**A:**
```
1. 给定初始提示词 → tokenize → 得到 idx (1, prompt_len)
2. 循环 max_new_tokens 次：
   a. 截取最后 context_length 个 token 作为输入
   b. model(idx_cond) → logits
   c. 取最后一个位置的 logits → idx_next = argmax(logits[-1])
   d. 拼接到 idx 末尾
3. 解码 idx → 文本
```

</details>

<details>
<summary><b>Q17: 训练集和验证集的困惑度（Perplexity）趋势说明了什么？</b></summary>

**A:**
- **两者同时下降** → 模型在学习（正常）
- **训练损失下降，验证损失上升** → **过拟合**
- **训练损失不降** → 学习率太低或模型有问题
- **验证损失远高于训练损失** → 训练数据太少或不具代表性

</details>

### 📇 第 6 章：分类微调

<details>
<summary><b>Q18: 分类微调时，GPT 模型做了什么修改？</b></summary>

**A:**
1. 加载预训练 GPT 模型权重
2. **替换输出头**：将 `out_head`（vocab_size 分类）替换为新的分类层（2 分类或 N 分类）
   ```python
   model.out_head = nn.Linear(emb_dim, num_classes)
   ```
3. 冻结部分层（可选），只训练最后几个 Transformer 块 + 新分类头
4. 通常在最后一个 token 位置取输出作分类预测（因为 causal mask 让它聚合了全局信息）

</details>

<details>
<summary><b>Q19: 微调时为什么通常只训练最后几层而非全部？</b></summary>

**A:**
- 底层学的是通用语言特征（语法、词汇），具有迁移性
- 顶层学的是任务特定特征
- 只训练顶层：参数少、训练快、不易过拟合、显存需求低
- 实验表明：训练最后一个 transformer block 通常就够用

</details>

### 📇 第 7 章：指令微调

<details>
<summary><b>Q20: 指令微调和分类微调的本质区别是什么？</b></summary>

**A:**

| 维度 | 分类微调 | 指令微调 |
|------|---------|---------|
| 输出 | 单个类别标签 | 完整文本回复 |
| 训练目标 | 交叉熵（分类） | 交叉熵（生成，逐 token） |
| 数据格式 | `(文本, 标签)` | `(指令, 输入, 期望输出)` |
| 模型修改 | 替换输出头 | 不修改架构 |
| 用途 | 特定任务 | 通用对话 |

</details>

<details>
<summary><b>Q21: 什么是对齐（Alignment）？DPO 和 RLHF 的区别？</b></summary>

**A:**

**对齐**：让模型输出符合人类偏好（有用、无害、诚实），而不是仅仅准确。

| 方法 | 原理 | 特点 |
|------|------|------|
| RLHF | 训练奖励模型 + PPO 强化学习 | 经典方法，训练复杂 |
| DPO | 直接在偏好数据对上优化 | 无需奖励模型，实现简单 |

DPO 核心思想：好回复的概率应该比坏回复高，优化目标是这个概率比值的对数。

</details>

### 📇 附录补充

<details>
<summary><b>Q22: LoRA 如何实现参数高效微调？</b></summary>

**A:**

冻结原始权重矩阵 W，在旁路添加可训练的低秩矩阵 A、B：

$$W' = W + \Delta W = W + A \cdot B$$

其中 `A ∈ R^(d×r)`, `B ∈ R^(r×d)`, `r << d`（通常 r=8~64）。

**优势**：只训练 A 和 B，参数量极少（<1% 原始参数），显存需求大幅降低。

</details>

<details>
<summary><b>Q23: KV Cache 为什么能加速推理？</b></summary>

**A:**

无缓存时，每次生成新 token 都要对所有历史 token 重新计算 K 和 V（O(n²) 复杂度）。

KV Cache 将已计算的 K、V 存储起来，每步只需计算新 token 的 K、V（O(n) 复杂度）。

**代价**：显存随序列长度线性增长。

</details>

<details>
<summary><b>Q24: GQA（分组查询注意力）如何减少显存？</b></summary>

**A:**

MHA（标准）中，每个注意力头都有独立的 K、V。GQA 让多个 query 头共享一组 K、V。

```
MHA:  Q₁,Q₂,Q₃,Q₄ 各自对应 K₁,K₂,K₃,K₄
GQA:  Q₁,Q₂ 共享 K₁   |   Q₃,Q₄ 共享 K₂
```

K/V 数量从 `n_heads` 降为 `n_kv_groups`（如 Llama 3 中 n_heads=32, n_kv_groups=8），KV 缓存显存减少 75%。

</details>

<details>
<summary><b>Q25: 混合专家模型（MoE）的核心思想是什么？</b></summary>

**A:**

将 FFN 层替换为多个"专家"子网络。每个 token 通过路由器（router/gate）动态选择 top-k 个专家激活：

```python
router_logits = gate(x)          # 每个专家的得分
weights = softmax(top_k(router_logits))  # 选 top-k 并归一化
output = sum(w_i * expert_i(x))  # 加权求和
```

**效果**：模型总参数量很大，但每次推理只激活一小部分，计算量可控。

</details>
