## 二、核心知识点

### 2.1 Python 版本选择

**原则：不要修改系统自带的 Python。**

操作系统（macOS/Linux/Windows）通常预装了 Python，但这个 Python **是给系统用的**，直接用它安装包可能会搞坏系统依赖。

```bash
# 检查系统 Python 版本
python --version    # 可能 "command not found"
python3 --version   # macOS 可能显示 3.9（旧版本）
```

**推荐做法**：安装一个独立的新版本 Python。

| 推荐 | 原因 |
|------|------|
| **Python 3.11 或 3.12** | PyTorch 等科学计算库对新版 Python 支持滞后几个月 |
| Python 3.13 | 可以用，但个别包可能有兼容问题 |
| Python 3.10 | 最低推荐版本 |

> 本书的代码已在 3.10、3.11、3.12、3.13 上全部测试通过。

**安装方式**（任选一种）：
- 官方下载：[python.org](https://www.python.org/downloads/)
- macOS Homebrew：`brew install python@3.11`
- Linux：`sudo apt install python3.11`
- Windows：从 python.org 下载安装包

---

### 2.2 包管理器：pip / conda / UV

Python 有多个包管理器，速度和体验差异很大：

| 工具 | 类型 | 速度 | 推荐度 |
|------|------|------|--------|
| **UV** | 新一代，Rust 编写 | ⚡ 最快 | ⭐⭐⭐⭐⭐ |
| pip | Python 官方内置 | 🐢 一般 | ⭐⭐⭐ |
| conda/miniforge | 科学计算全家桶 | 🐢 较慢 | ⭐⭐⭐ |

**本书推荐 UV**，原因：
1. **极快** — 比 pip 快 10-100 倍
2. **pip 兼容语法** — `uv pip install` 和 `pip install` 几乎一样
3. **Python 社区新宠** — 大量开发者已迁移

**安装 UV**：
```bash
# 方法一：用 pip 安装（需要已有 Python）
pip install uv

# 方法二：独立安装（不需要 Python）
# macOS/Linux
curl -LsSf https://astral.sh/uv/install.sh | sh
# Windows
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

UV 有两种用法：
```bash
# uv pip 模式（本教程使用，对 pip 用户最友好）
uv pip install torch
uv pip install -r requirements.txt

# 原生 uv 模式（更现代，推荐进阶用户探索）
uv add torch        # 自动更新 pyproject.toml
uv sync             # 根据 lockfile 安装所有依赖
```

### 2.3 虚拟环境（Virtual Environment）

**虚拟环境 = 一个独立文件夹，内含自己的 Python 和所有包。**

```
LLMs-from-scratch/          ← 项目根目录
├── .venv/                  ← 虚拟环境（可随时删除重建）
│   ├── bin/
│   │   ├── python          ← 独立的 Python 3.11
│   │   ├── pip
│   │   └── ...
│   └── lib/
│       └── python3.11/
│           └── site-packages/  ← 所有安装的包
├── ch02/
├── ch03/
└── requirements.txt
```

**为什么用虚拟环境？**
- 🛡️ **隔离** — 项目之间互不干扰
- 🔄 **安全** — 搞坏了直接删除 `.venv` 重建，不影响系统
- 📋 **可复现** — `pip freeze > requirements.txt` 记录精确版本

**操作流程**（只需做一次）：

```bash
# 1. 创建虚拟环境（指定 Python 版本）
uv venv --python 3.11

# 2. 激活虚拟环境
# macOS/Linux:
source .venv/bin/activate
# Windows:
.venv\Scripts\activate

# 3. 验证使用的是虚拟环境中的 Python
which python    # macOS/Linux
where python    # Windows
# 应显示: .../LLMs-from-scratch/.venv/bin/python
```

> ⚠️ 每次打开新终端都要重新激活虚拟环境（`source .venv/bin/activate`）。

### 2.4 安装依赖

本书根目录有 `requirements.txt`，包含所有必需的包：

| 包 | 版本要求 | 使用章节 | 说明 |
|----|---------|---------|------|
| `torch` | >= 2.2.2 | 全部 | PyTorch — 本书的深度学习框架。Intel macOS 有特殊版本限制 |
| `jupyterlab` | >= 4.0 | 全部 | Jupyter Notebook 运行环境 |
| `tiktoken` | >= 0.5.1 | Ch2, Ch4, Ch5 | OpenAI 的 BPE 分词器 |
| `matplotlib` | >= 3.7.1 | Ch4, Ch6, Ch7 | 数据可视化（loss 曲线等） |
| `tensorflow` | >= 2.18.0 | Ch5, Ch6, Ch7 | 仅用于加载 GPT-2 预训练权重（TensorFlow 格式） |
| `tqdm` | >= 4.66.1 | Ch5, Ch7 | 训练进度条 |
| `numpy` | >= 1.26 | 底层依赖 | 数值计算（torch/pandas 等会自动安装） |
| `pandas` | >= 2.2.1 | Ch6 | 数据处理（分类微调数据集） |
| `psutil` | >= 5.9.5 | Ch7 | 系统监控（torch 会自动安装） |

> 💡 注意 `requirements.txt` 中包含平台标记（Intel macOS 有特殊的 `torch` 和 `tensorflow` 版本约束），`pip`/`uv` 会自动处理，无需手动干预。

**一键安装**：
```bash
# 激活虚拟环境后执行（在项目根目录下）
uv pip install -r requirements.txt
```

**PyTorch GPU 用户**：如果你有 NVIDIA GPU 并想启用 CUDA 加速，建议先参考 [pytorch.org](https://pytorch.org) 的安装指南选择对应的 CUDA 版本，再安装其余依赖。第 2-4 章不需要 GPU，可以先跳过此步。

**Windows 用户注意**：TensorFlow 在 Windows 上可能遇到 `tensorflow-io-gcs-file-system` 问题。解决方案：
1. 从 `requirements.txt` 中删除 `tensorflow` 那一行，先安装其他包
2. 等到第 5 章需要时再单独处理 TensorFlow
3. 或者直接用 Google Colab 运行第 5 章

## 三、动手练习：从零配置环境

按以下步骤操作，确保每一步都成功再继续：

## 四、常见问题

**Q: 虚拟环境激活后命令行前面多了 `(.venv)`，正常吗？**
A: 正常。这个前缀表示你正在虚拟环境中。输入 `deactivate` 可以退出。

**Q: `python` 命令找不到，只有 `python3`？**
A: macOS/Linux 上通常需要用 `python3`。激活虚拟环境后，`python` 命令会指向虚拟环境中的 Python。

**Q: UV 安装包时报错怎么办？**
A: 回退使用 `pip install -r requirements.txt`，功能相同只是慢一些。

**Q: TensorFlow 安装失败怎么办？**
A: 本书只在第 5 章加载 GPT-2 权重时用到 TensorFlow。可以先跳过，等到第 5 章再处理。或直接用 Google Colab 运行第 5 章。

**Q: 没有 GPU 能学吗？**
A: 完全可以。第 2-4 章不需要 GPU，普通笔记本即可。第 5 章训练用 Colab 免费 T4。

**Q: Docker 用户怎么配置？**
A: 参见 `setup/03_optional-docker-environment/README.md`，有完整的 DevContainer 配置指南。
