## 分类微调 vs 指令微调

| 特性 | 分类微调 (ch06) | 指令微调 (ch07) |
|------|----------------|----------------|
| 目标 | 文本分类 (spam/ham) | 生成式回复 |
| 输出层 | 2 classes (替换) | vocab_size (保留) |
| 训练数据 | (文本, 标签) 对 | (指令, 回答) 对 |
| 使用 | 最后 token 的 logits | 所有 token 的 logits |

## 模块 2: 模型修改

### 替换输出头

```python
# 原始 GPT-2 (语言模型)
model.out_head = Linear(768, 50257)  # → 词汇表大小

# 分类微调 (替换为 2 类)
model.out_head = Linear(768, 2)      # → spam / ham
```

### 冻结策略

```python
# 冻结所有参数
for param in model.parameters():
    param.requires_grad = False

# 只解冻最后一层 Transformer + final_norm + out_head
for param in model.trf_blocks[-1].parameters():
    param.requires_grad = True
for param in model.final_norm.parameters():
    param.requires_grad = True
for param in model.out_head.parameters():
    param.requires_grad = True
```

### 参数量对比

| 组件 | 参数量 | 训练状态 |
|------|--------|---------|
| 前 11 层 Transformer | ~78M | ❄️ 冻结 |
| 第 12 层 Transformer | ~7.1M | 🔥 训练 |
| final_norm | 1,536 | 🔥 训练 |
| out_head (新) | 1,538 | 🔥 训练 |
| Token/Pos Embedding | ~39M | ❄️ 冻结 |
| **可训练总计** | **~7.1M** | **4.3%** |

### 为什么只用最后一个 token？

```
因果注意力 (Causal Attention):
  token 0: 只看自己
  token 1: 看 0, 1
  ...
  token N: 看 0, 1, ..., N  ← 看到所有 token!

→ 最后一个 token 的输出 = 整个句子的"摘要"
→ logits[:, -1, :] 就是分类依据

对比:
  token 0    → 只包含第一个词的信息
  token N-1  → 包含整个句子的信息 ✓
```

## 模块 3: 损失函数与准确率

### 分类 Loss (与 ch05 不同)

```python
# ch05 (语言模型): 使用所有 token 的 logits
loss = cross_entropy(logits.flatten(0,1), targets.flatten())

# ch06 (分类): 只用最后一个 token 的 logits
loss = cross_entropy(logits[:, -1, :], targets)
#                     ↑ 只取最后位置
```

### 准确率计算

```python
def calc_accuracy(loader, model, device):
    correct, total = 0, 0
    for input_batch, target_batch in loader:
        logits = model(input_batch)[:, -1, :]
        predictions = torch.argmax(logits, dim=-1)
        correct += (predictions == target_batch).sum().item()
        total += target_batch.numel()
    return correct / total
```

## 模块 5: 推理与部署

### 分类函数

```python
def classify_review(text, model, tokenizer, device, max_length, pad_token_id=50256):
    model.eval()
    input_ids = tokenizer.encode(text)
    input_ids = input_ids[:max_length] + [pad_token_id] * (max_length - len(input_ids))
    input_tensor = torch.tensor(input_ids).unsqueeze(0).to(device)

    with torch.no_grad():
        logits = model(input_tensor)[:, -1, :]
    predicted_label = torch.argmax(logits, dim=-1).item()

    return "spam" if predicted_label == 1 else "not spam"
```

## 核心架构图

```
输入: "You are a winner receive $1000!"
  ↓ tiktoken
Token IDs: [1639, 466, 318, ...]
  ↓ padding to 120
[1639, 466, 318, ..., 50256, 50256]
  ↓ GPT-2 124M (11 层冻结 + 1 层训练)
Embeddings: (120, 768)
  ↓ Transformer × 12
Hidden states: (120, 768)
  ↓ final_norm
  ↓ 只取最后一个 token: hidden_states[:, -1, :]
(768,)
  ↓ out_head: Linear(768 → 2)
Logits: [ham_score, spam_score]
  ↓ argmax
预测: spam (1) 或 ham (0)
```

## 关键决策总结

| 决策 | 选择 | 原因 |
|------|------|------|
| 替换输出层 | 50257 → 2 | 分类任务只需类别数 |
| 冻结策略 | 解冻最后 1 层 | 平衡效果与效率 |
| 分类 token | 最后一个 | 因果注意力 = 全局摘要 |
| Loss 函数 | cross_entropy(logits[:,-1,:], target) | 只在最后位置计算 |
| 优化器 | AdamW (lr=5e-5) | 比预训练更小的学习率 |
