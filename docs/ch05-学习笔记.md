## 模块 1: Cross-Entropy Loss 推导

### 从概率到 Loss 的完整链

```
模型输出 logits  (batch, seq, vocab)
     ↓ softmax
概率分布 probas  (batch, seq, vocab)
     ↓ 取 target 位置的概率
target 概率      (batch, seq)
     ↓ log
对数概率         (batch, seq)
     ↓ mean × (-1)
Cross-Entropy Loss ← 最小化这个值
     ↓ exp
Perplexity (困惑度)
```

### 实践数据

| 指标 | 训练前 | 含义 |
|------|--------|------|
| target 概率 | ~0.00002 | 模型给正确词的概率极低 |
| log 概率均值 | -10.79 | 取对数后仍很小 |
| CE Loss | 10.79 | 标准交叉熵 |
| Perplexity | 48,726 | 模型在 ~49K 个词之间犹豫 |

**PyTorch 一步完成：**`F.cross_entropy(logits.flatten(0,1), targets.flatten())`

```
val_loss:   10.98 → 6.38  (早期下降后停滞)

→ 原因: 数据集太小 (5K tokens), 模型背下来了
→ 解决: 用更大数据集 或 加载预训练权重
```

## 模块 3: 加载 OpenAI 预训练权重

### 权重映射

```
OpenAI 格式 (TensorFlow)          →  我们的 PyTorch 格式

params["wte"]  (50257, 768)      →  model.tok_emb.weight
params["wpe"]  (1024, 768)       →  model.pos_emb.weight
blocks[b]["attn"]["c_attn"]["w"] →  分拆为 W_q, W_k, W_v (各 768×768)
blocks[b]["attn"]["c_proj"]["w"] →  att.out_proj.weight
blocks[b]["mlp"]["c_fc"]["w"]    →  ff.layers[0].weight
blocks[b]["mlp"]["c_proj"]["w"]  →  ff.layers[2].weight
blocks[b]["ln_1"]["g"]           →  norm1.scale (γ)
blocks[b]["ln_1"]["b"]           →  norm1.shift (β)
params["g"]                      →  final_norm.scale
params["b"]                      →  final_norm.shift
params["wte"]                    →  out_head.weight (权重绑定!)
```

### 关键细节

1. **qkv_bias=True**: OpenAI GPT-2 的 Q/K/V 有偏置
2. **c_attn 合并存储**: OpenAI 把 Q/K/V 存在一起 (768, 2304)，需 `np.split` 拆开
3. **权重转置**: OpenAI 存的是 `(in, out)`，PyTorch Linear 要 `(out, in)`，需 `.T`
4. **Weight tying**: `out_head.weight = tok_emb.weight`

### 生成结果对比

| 来源 | 输出 | 评价 |
|------|------|------|
| 随机权重 | "Featureiman Byeswickattribute..." | 无意义 |
| 小数据训练 | "moves you? Yes--quite insensible..." | 语法对但过拟合 |
| **OpenAI 预训练** | **"moves you forward. The first step is..."** | **流畅连贯** |

## 模块 4: 解码策略

### 三种策略

```
1. 贪婪解码 (temp=0)
   argmax(logits) → 每次选概率最高的词
   确定性、快速，但缺乏多样性

2. 温度采样 (temp>0)
   softmax(logits / temperature) → 按概率采样
   temp < 1: 更尖锐 (更确定)
   temp > 1: 更平坦 (更多样)

3. Top-K 采样
   只在概率最高的 K 个词中采样
   过滤掉低质量的长尾词
   通常与温度采样组合使用
```

### 实践输出

```
[贪婪解码] temp=0.0
  Every effort moves you forward.
  The first step is to understand the importance of your work.

[温度采样] temp=0.8
  Every effort moves you forward and you work hard to achieve your dreams.

[温度+TopK] temp=0.8, top_k=5
  Every effort moves you forward and you will be able to achieve more.
```

### 温度公式

```python
softmax(logits / temperature)

  temperature → 0:  趋近 argmax (贪婪)
  temperature = 1.0: 标准 softmax
  temperature → ∞:  趋近均匀分布
```

### Top-K 实现

```python
# 只保留 top-k 个 logits, 其余设为 -inf
top_logits, _ = torch.topk(logits, k)
logits = torch.where(
    logits < top_logits[:, -1],
    torch.full_like(logits, float('-inf')),
    logits
)
probas = torch.softmax(logits, dim=-1)
idx_next = torch.multinomial(probas, num_samples=1)
```

## 模块 5: 模型保存与加载

### 基本保存

```python
# 保存
torch.save(model.state_dict(), "model.pth")

# 加载
model = GPTModel(cfg)
model.load_state_dict(torch.load("model.pth"))
```

### 完整 checkpoint (含优化器状态)

```python
# 保存
torch.save({
    "model_state_dict": model.state_dict(),
    "optimizer_state_dict": optimizer.state_dict(),
}, "model_and_optimizer.pth")

# 加载
checkpoint = torch.load("model_and_optimizer.pth")
model.load_state_dict(checkpoint["model_state_dict"])
optimizer.load_state_dict(checkpoint["optimizer_state_dict"])
```
