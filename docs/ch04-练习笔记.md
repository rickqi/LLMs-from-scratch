## 练习 4.1: FeedForward vs Attention 参数量对比

### 问题

单个 TransformerBlock 中，FeedForward 和 Attention 各占多少参数？

### 答案

| 组件 | 计算式 | 参数量 |
|------|--------|--------|
| **FeedForward** | `Linear(768→3072)` + `Linear(3072→768)` | **4,722,432** |
| 第 1 层 | 768×3072 + 3072(bias) | 2,362,368 |
| 第 2 层 | 3072×768 + 768(bias) | 2,360,064 |
| | | |
| **Attention** | `W_q + W_k + W_v + out_proj` | **2,360,064** |
| W_query | 768×768 (no bias) | 589,824 |
| W_key | 768×768 (no bias) | 589,824 |
| W_value | 768×768 (no bias) | 589,824 |
| out_proj | 768×768 + 768(bias) | 590,592 |
| | | |
| **LayerNorm ×2** | 每个 768+768 | 3,072 |
| **总计 (单层)** | | **7,085,568** |

### 数学推导

```
设 emb_dim = d

FeedForward:  d × 4d + 4d × d = 8d²  (+ bias ≈ 8d²)
Attention:    3 × d × d + d × d = 4d²  (+ bias ≈ 4d²)

比值: FF / Attn = 8d² / 4d² = 2.0
```

**FFN 参数是 Attention 的 2 倍！** Transformer 中 2/3 的参数在 FFN 里。

→ 这正是 **MoE (Mixture of Experts)** 的动机：用稀疏激活的多个小 FFN 替代一个大 FFN，减少推理计算量但保持参数容量。

## 练习 4.2: 初始化更大的 GPT 模型

### GPT-2 系列配置

| 模型 | emb_dim | n_layers | n_heads | 总参数 | weight-tied | 大小 |
|------|---------|----------|---------|--------|-------------|------|
| **small** (124M) | 768 | 12 | 12 | 163M | 124M | 622 MB |
| **medium** (355M) | 1024 | 24 | 16 | 406M | 355M | 1.5 GB |
| **large** (774M) | 1280 | 36 | 20 | 838M | 774M | 3.2 GB |
| **XL** (1558M) | 1600 | 48 | 25 | 1,638M | 1,557M | 6.2 GB |

### 参数增长规律

```
small  (d=768,  L=12):  163M  ████
medium (d=1024, L=24):  406M  ██████████      (~2.5×)
large  (d=1280, L=36):  838M  ████████████████████ (~2×)
XL     (d=1600, L=48): 1638M  ████████████████████████████████████████ (~2×)
```

- emb_dim 增大 → 每层参数 ∝ d²
- n_layers 增大 → 总参数 ∝ L
- 组合效果: 参数 ∝ L × d²

### Weight Tying 的影响

```
out_head: emb_dim × vocab_size

  small: 768 × 50257  = 38.6M  (占总参数 24%!)
  XL:    1600 × 50257 = 80.4M   (占总参数 5%)
```

模型越大，weight tying 省的占比越小（因为主体 trf_blocks 更大）。

### AMD 890M 适用性

```
✅ small  (622 MB)  — 完全可以跑
⚠️  medium (1.5 GB)  — 需要系统内存共享
❌ large  (3.2 GB)  — VRAM 不够
❌ XL     (6.2 GB)  — VRAM 不够
```

## 练习 4.3: 独立的 Dropout 参数

### 改动

```python
# 原配置: 一个 dropout 统管所有位置
{"drop_rate": 0.1}

# 新配置: 3 个独立 dropout
{
    "drop_rate_emb": 0.1,        # embedding 后的 dropout
    "drop_rate_attn": 0.1,       # attention weights 上的 dropout
    "drop_rate_shortcut": 0.1,   # 残差路径上的 dropout
}
```

### 代码修改点

1. **TransformerBlock.__init__**:
   - `MultiHeadAttention(dropout=cfg["drop_rate_attn"])` — 原来 `cfg["drop_rate"]`
   - `nn.Dropout(cfg["drop_rate_shortcut"])` — 原来 `cfg["drop_rate"]`

2. **GPTModel.__init__**:
   - `nn.Dropout(cfg["drop_rate_emb"])` — 原来 `cfg["drop_rate"]`

### 为什么需要独立参数？

| 位置 | Dropout 作用 | 调参方向 |
|------|-------------|---------|
| Embedding | 防止词嵌入过拟合 | 数据少时增大 |
| Attention | 正则化注意力权重 | 让模型少依赖某些 token |
| Shortcut | 防止残差路径过拟合 | 深层网络时可增大 |

### 现代趋势

```python
# 现代大模型 (Llama, GPT-4 等) 通常不用 dropout
{
    "drop_rate_emb": 0.0,
    "drop_rate_attn": 0.0,
    "drop_rate_shortcut": 0.0,
}
# 原因: 训练数据足够多, 靠数据本身防过拟合, 不需要 dropout
```

### GPT-2 124M 中的 Dropout 总数

```
model.drop_emb:          1 个
12 × TransformerBlock:
  block.att.dropout:     1 个 × 12 = 12
  block.drop_shortcut:   2 个 × 12 = 24  (attention + FFN 各一个)
────────────────────────────────────
总计:                    37 个 Dropout 层
```
