## 方案 B: Qwen2.5-0.5B + LoRA (推荐)

### 模型信息

```
模型:   Qwen/Qwen2.5-0.5B (通过 hf-mirror.com 下载)
参数:   494M (24 layers, 14 heads, hidden=896)
架构:   qwen2 (RoPE + RMSNorm + SwiGLU + GQA)
微调:   LoRA (rank=8, target: q_proj/k_proj/v_proj)

```

### Tokenizer 效率对比 (核心优势)

```
文本: "临床表现：患者出现持续性腹痛，伴有恶心呕吐。"

GPT-2:  52 tokens  ← 中文被拆成字节碎片
Qwen:   14 tokens  ← "临床" = 1 token, "表现" = 1 token
压缩比: 3.7×  ← Qwen tokenizer 对中文的压倒性优势

| 文本                     | GPT-2 | Qwen | 压缩比 |
|--------------------------|-------|------|--------|
| 临床表现：患者出现持续... |    52 |   14 |   3.7× |
| 诊断标准：根据病理学...   |    54 |   13 |   4.2× |
| 治疗方法：首选手术切...   |    53 |   14 |   3.8× |

编码对比 ("肿瘤的诊断"):
  GPT-2: [47553, 102, 114, 165, 16435, 102, 114, 31085, 102, 114, 3142]  (11 tokens)
  Qwen:  [805, 30442, 31546, 31085]  (4 tokens)  ← "肿瘤" 是 1 个 token!
```

### LoRA 微调配置

```python
from peft import LoraConfig, get_peft_model, TaskType

lora_config = LoraConfig(

    lora_alpha=16,          # 缩放因子

model = get_peft_model(model, lora_config)
# 可训练参数: ~0.5M / 494M = 0.1%
```

## 数据分析

| 指标 | 数值 |
|------|------|
| 总分册数 | 77 个 .md 文件 |
| 总文本量 | ~15M 字符 (清洗后) |
| 选中训练 | 30K 字符 (肿瘤分册子集) |
| Token 数 | ~55K (GPT-2 tokenizer) |

### 关键发现：GPT-2 tokenizer 对中文效率极低

```
英文: 1 token ≈ 4 字符  (高效)
中文: 1 token ≈ 0.7 字符 (低效! 1个中文字需要1.4个token)

原因: GPT-2 的 BPE 词表是为英文设计的
      中文字符被拆分成 UTF-8 字节级 token
      "肿" → [33269, 102, 114] = 3 tokens!
```

## 训练流程

```
1. 数据清洗
   - 去除 HTML 标签、图片标记、ISBN
   - 去除 OCR 残留 (page=bbox=...)
   - 去除多余空行

2. Tokenize
   text → tiktoken (GPT-2 BPE) → token IDs

3. 滑动窗口
   create_dataloader_v1(batch=4, ctx=64, stride=64)

4. 继续预训练
   - 加载 OpenAI GPT-2 预训练权重
   - AdamW(lr=5e-4, weight_decay=0.1)
   - 50 batches (~80s on CPU)
```

## 训练结果

| 阶段 | Loss | 生成样本 |
|------|------|---------|
| 训练前 | ~10 | `临床表现：＝＝＝＝＝` (完全乱码) |
| 50 batches | 3.1 | `临床表现：组疗。` (开始出现中文字) |

### 训练前 vs 后

```
训练前: GPT-2 对中文一窍不通 → 输出 "=" 符号
训练后: 开始学习中文模式 → 输出 "组疗" 等医学相关字符
```

## 为什么质量有限？

1. **Tokenizer 不匹配**: GPT-2 BPE 为英文设计，中文 1 字 = 1.4 tokens
2. **训练太少**: 仅 50 batches（正式训练需要 1000+ batches）
3. **预训练权重是英文**: GPT-2 从未见过中文，需要大量训练才能学到
4. **模型太小**: 124M 参数对中文生成力不足

## 中文 LLM 正确方案

```
本项目 (GPT-2 中文):  ❌ tokenizer 不匹配, 预训练无中文知识
正确方案 (Qwen 3 等):  ✅ 中文 tokenizer + 中文预训练权重

  GPT-2:  50257 vocab, 英文 BPE, 0.7 中文字/token
  Qwen 3: 151936 vocab, 中英双语 BPE, ~2 中文字/token
  ChatGLM: 130528 vocab, 中英双语, ~2 中文字/token

  → 中文项目应该用 Qwen / ChatGLM / DeepSeek
  → 本项目展示了 pipeline, 但 GPT-2 不适合中文
```

## 完整 Pipeline 代码

```python
# 1. 数据清洗
def clean_medical_text(text):
    text = re.sub(r'<[^>]+>|\!\[.*?\]\(.*?\)|ISBN[^\n]*', '', text)
    text = re.sub(r'page=\d+[^)]*\)', '', text)
    text = re.sub(r'\n{4,}', '\n\n', text)
    return '\n'.join(l.strip() for l in text.split('\n') if l.strip())

# 2. DataLoader
train_loader = create_dataloader_v1(
    cleaned_text, batch_size=4, max_length=64,
    stride=64, drop_last=True, shuffle=True)

# 3. 加载预训练模型
model = GPTModel(GPT_CONFIG_124M)
model.load_state_dict(torch.load("gpt2_pretrained.pth"))

# 4. 继续预训练
optimizer = AdamW(model.parameters(), lr=5e-4, weight_decay=0.1)
for input_batch, target_batch in train_loader:
    optimizer.zero_grad()
    loss = F.cross_entropy(
        model(input_batch).flatten(0,1),
        target_batch.flatten())
    loss.backward()
    optimizer.step()

# 5. 生成
ids = tokenizer.encode("临床表现：")
for _ in range(30):
    logits = model(torch.tensor([ids])[:, -64:])
    ids.append(torch.argmax(logits[:, -1, :]).item())
print(tokenizer.decode(ids))
```

## 改进方向

1. **更长训练**: 更多 epochs + 更大数据 (全部 15M 字符) → 更专业的医学文本
2. **指令微调**: 构建医学 Q&A 对 (症状→诊断→治疗), 做问答助手
3. **RAG 方案**: 不微调, 用检索增强生成 + 医学知识库 (最适合知识密集型)
4. **更大模型**: Qwen2.5-7B 或 Qwen3-8B (需要 GPU ≥16GB)
5. **多模态**: 加入医学影像 (X光/CT) 做多模态诊断辅助
6. **评估**: 用专业医学考试题 (如执业医师资格考试) 评估模型能力
