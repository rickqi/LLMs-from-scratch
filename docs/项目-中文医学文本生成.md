# 实战项目：中文医学诊疗指南文本生成

> 方案 A: GPT-2 124M 继续预训练（效果有限，仅作教学演示）
> 方案 B: Qwen3-0.6B + LoRA 微调（推荐方案）

---

## 为什么选择 Qwen3-0.6B

≤1B 参数范围内，Qwen3-0.6B 是**唯一具备实用中文生成能力的模型**：

| 模型 | 参数 | 词表 | 中文效率 | 预训练数据 | 推荐度 |
|------|------|------|---------|-----------|--------|
| **Qwen3-0.6B** | 600M | 151K | 1.6 字/token | 36T, 中英双语 | ✅ 唯一最优 |
| Qwen2.5-0.5B | 494M | 151K | 1.6 字/token | 18T, 中英双语 | ✅ 可用 |
| SmolLM2-360M | 360M | 49K | 0.5 字/token | ~4T, 英文为主 | ❌ 不识中文 |
| Llama-3.2-1B | 1,000M | 128K | 0.5 字/token | ~9T, 英文为主 | ❌ 不识中文 |
| MiniMind | 26-64M | 6K | ~1.0 字/token | 少量中文 | ❌ 玩具模型 |

**核心原因**: 英文模型的 BPE 词表几乎不包含中文词汇 → 1个中文字被拆成 3-5 个 token → 相当于模型容量缩水 3-5 倍。

## 方案 B: Qwen3-0.6B + LoRA

### 模型信息

```
模型:   Qwen/Qwen3-0.6B (通过 hf-mirror.com 下载)
```

### LoRA 配置

```python
from peft import LoraConfig, get_peft_model, TaskType

lora_config = LoraConfig(
    task_type=TaskType.CAUSAL_LM,
    r=8,
    lora_alpha=16,
    lora_dropout=0.05,
    target_modules=["q_proj", "k_proj", "v_proj", "o_proj"],
)
model = get_peft_model(model, lora_config)
# 可训练参数: ~0.7M / 600M = 0.12%
```

## Qwen3-0.6B vs Qwen2.5-0.5B 升级分析

| 维度 | Qwen2.5-0.5B | Qwen3-0.6B | 提升 |
|------|-------------|-----------|------|
| 参数 | 494M | 600M | +21% |
| 层数 | 24 | 28 | +17% |
| 注意力 | 14头 MHA | 16Q+8KV GQA | 更高推理效率 |
| 隐藏维 | 896 | 1024 | +14% |
| 预训练数据 | 18T tokens | **36T tokens** | 翻倍 |
| 训练阶段 | 2 阶段 | 3 阶段 (含长文训练) | 更充分 |
| 医学微调 | 有实践 | **已有论文验证** | Dr. Qwen |
| 推理显存(LoRA) | ~3.5GB | ~4.0GB | 仍可消费级 |
| 许可 | Apache 2.0 | Apache 2.0 | 相同 |

## 方案 A: GPT-2 124M (仅作参考)

### 项目概要

```
数据源: 临床诊疗指南 — 肿瘤分册 (中华医学会)
模型:   GPT-2 124M (预训练权重)
任务:   继续预训练 (continued pretraining)
目标:   让模型学习中文医学文本的语言模式
```

### 关键发现：GPT-2 tokenizer 对中文效率极低

```
英文: 1 token ≈ 4 字符  (高效)
中文: 1 token ≈ 0.7 字符 (低效! 1个中文字需要1.4个token)

原因: GPT-2 的 BPE 词表是为英文设计的
      中文字符被拆分成 UTF-8 字节级 token
      "肿" → [33269, 102, 114] = 3 tokens!
```

### 训练流程

```
1. 数据清洗 → 去 HTML/ISBN/OCR 残留
2. Tokenize → tiktoken (GPT-2 BPE)
3. 滑动窗口 → create_dataloader_v1(batch=4, ctx=64, stride=64)
4. 继续预训练 → AdamW(lr=5e-4), 50 batches
```

### 训练结果

| 阶段 | Loss | 生成样本 |
|------|------|---------|
| 训练前 | ~10 | `临床表现：＝＝＝＝＝` (完全乱码) |
| 50 batches | 3.1 | `临床表现：组疗。` (开始出现中文字) |

## 完整执行计划

详见 `projects/chinese-medical-text-generation/ANALYSIS.md`，核心路径：

| 阶段 | 内容 | 产出 |
|------|------|------|
| 环境 | pip install -r requirements.txt | 就绪 |
| 数据 | data_prep.py (真实数据 / 样本数据) | train.txt + val.txt |
| 训练 | train_qwen_lora.py (Qwen3-0.6B + LoRA) | LoRA 权重 + training_log.json |
| 评估 | generate.py | 医学文本生成样本 |
| 迭代 | 增数据 / 调参 / 换大模型 | 持续提升 |

## 改进方向

1. **数据扩充**: 全部 77 册 (15M → 50M+ 字符)
2. **指令微调**: 构建医学 QA 对 (症状→诊断→治疗), 做问答助手
3. **RAG 方案**: 不微调, 用检索增强生成 + 医学知识库
4. **更大模型**: Qwen3-8B (需要 GPU ≥16GB)
5. **多模态**: 加入医学影像 (X光/CT) 做多模态诊断辅助
6. **评估体系**: 用执业医师资格考试题评估模型能力
