## 模块 2: Layer Normalization

### 为什么不用 BatchNorm？

| 特性 | BatchNorm | LayerNorm |
|------|-----------|-----------|
| 归一化维度 | 沿 batch 维度 | 沿特征维度 |
| 依赖 batch size | 是（小 batch 不稳定） | 否 |
| 推理时 | 需要 running stats | 不需要 |
| 适合序列模型 | 否 | **是** |

### 公式

```
LayerNorm(x) = γ · (x - μ) / √(σ² + ε) + β

  μ = x.mean(dim=-1)      # 沿特征维度求均值
  σ² = x.var(dim=-1)       # 沿特征维度求方差
  γ, β = 可学习参数         # 每个特征维度一对
  ε = 1e-5                 # 防止除以 0
```

### 实现要点

- `unbiased=False`：使用有偏方差估计（除以 N 而非 N-1）
- γ 初始化为 1，β 初始化为 0（初始时等价于纯归一化）
- 每层 2 × emb_dim 个参数

## 模块 3: FeedForward 网络 (GELU)

### 结构

```
Linear(768 → 3072)    ← 扩展 4 倍
GELU()                ← 平滑激活
Linear(3072 → 768)    ← 压缩回来

参数量: 768×3072 + 3072 + 3072×768 + 768 = 4,722,432 (~4.7M per layer)
```

### GELU vs ReLU

| 特性 | ReLU | GELU |
|------|------|------|
| 公式 | max(0, x) | 0.5x(1 + tanh(√(2/π)(x + 0.044715x³))) |
| 负值处理 | 硬截断归零 | 平滑过渡（保留部分负值信息） |
| x=0 处 | 不可导 | 处处可导 |
| 梯度 | 不连续 | 平滑 |
| 使用 | 早期 CNN | **GPT/BERT/现代 LLM** |

### 为什么 FeedForward？

- Attention 负责**词间信息混合**（全局）
- FeedForward 负责**词内特征变换**（局部，每个 token 独立处理）
- 两者互补：Attention 看全局关系，FFN 做非线性变换

## 模块 4: 残差连接 (Residual Connections)

### 核心公式

```
x_output = x_input + f(x_input)

梯度链: d(loss)/d(x_input) = d(loss)/d(x_output) · (1 + f'(x_input))
                                                       ↑↑↑
                                            "+1" 保证梯度不会消失
```

### 实验对比

| 层 | 无残差梯度 std | 有残差梯度 std |
|----|---------------|---------------|
| layer.0 | 0.0025 | **0.1832** |
| layer.1 | 0.0007 | **0.1981** |
| layer.2 | 0.0083 | **0.2330** |
| layer.3 | 0.0079 | **0.1021** |
| layer.4 | 0.0534 | **0.6237** |

**结论**：残差连接使早期层梯度从 0.002 → 0.18，**提升 70 倍**！

### 在 Transformer 中的应用

```python
# Attention 子层
x = x + dropout(self_att(layer_norm(x)))     # Pre-LN + 残差

# FeedForward 子层
x = x + dropout(ffn(layer_norm(x)))          # Pre-LN + 残差
```

**Pre-LN**（先 LayerNorm 再进子层）是 GPT 的选择，比原始 Transformer 的 Post-LN 训练更稳定。

## 模块 5: Transformer Block

```
输入 x  (batch, tokens, 768)
  │
  ├── shortcut ──────────────────┐
  ▼                              │
LayerNorm(768)                   │
  │                              │
  ▼                              │

  │                              │
  ▼                              │
Dropout(0.1)                     │
  │                              │
  ▼                              │
  + ←────────────────────────────┘  残差连接 1
  │

  ▼                              │

  │                              │
  ▼                              │

  │                              │

```

- 12 层总计：~85M（GPT-2 124M 的主体）

## 模块 6: 完整 GPT 模型

### 参数量分析

| 组件 | 参数量 | 说明 |
|------|--------|------|
| Token Embedding | 38.6M | 50257 × 768 |
| Position Embedding | 0.8M | 1024 × 768 |
| 12 × TransformerBlock | ~85M | 主体 |
| Final LayerNorm | 1,536 | 可忽略 |
| Output Head (Linear) | 38.6M | 768 × 50257 |
| **总计** | **163M** | |

**为什么不是 124M？**
原始 GPT-2 使用**权重绑定**（weight tying）：`out_head.weight = tok_emb.weight`，省掉 38.6M → 124M。本书没用权重绑定，方便训练。

## 模块 7: 文本生成（贪婪解码）

### 算法

```
for each new token:
    1. 取最后 context_size 个 token 作为输入
    2. 模型前向传播 → logits (batch, seq, vocab)
    3. 只取最后一个位置的 logits → (batch, vocab)
    4. softmax → 概率分布
    5. argmax → 选概率最高的词 (贪婪, 不采样)
    6. 拼接到序列末尾
```

### 贪婪 vs 采样

| 方法 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| 贪婪 (argmax) | 每步选最高概率词 | 确定性、快速 | 缺乏多样性，可能重复 |
| 温度采样 | 按概率分布采样 | 多样性强 | 可能产生低质量文本 |
| Top-k 采样 | 只在 top-k 中采样 | 平衡质量和多样性 | ch05 详述 |
