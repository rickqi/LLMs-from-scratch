## 模块 2: Custom Collate + -100 Masking

### 核心概念：-100 Masking

```python
# PyTorch cross_entropy 自动忽略 target = -100 的位置
loss = F.cross_entropy(logits, targets)  # targets 中 -100 不参与计算
```

### 为什么需要 -100？

```
输入序列: [inst1, inst2, ..., resp1, resp2, resp3, pad, pad]

如果不 mask:
  loss 在所有位置计算 → 模型学习 "预测 padding" (浪费!)

用 -100 mask 后:
  target: [-100, -100, ..., resp1, resp2, resp3, -100, -100]
                            ↑ 只在这些位置计算 loss
```

### Custom Collate 函数

```python
def custom_collate_fn(batch, pad_token_id=50256, ignore_index=-100):
    batch_max_length = max(len(item) for item in batch)

    inputs_lst, targets_lst = [], []
    for item in batch:
        # Step 1: Padding 到批次最长
        padded = item + [pad_token_id] * (batch_max_length - len(item))

        # Step 2: 输入 = 原序列, 目标 = 右移一位
        inputs = torch.tensor(padded)
        targets = torch.tensor(padded[1:] + [pad_token_id])

        # Step 3: 将目标中的 pad_token_id 替换为 -100
        targets[targets == pad_token_id] = ignore_index

        inputs_lst.append(inputs)
        targets_lst.append(targets)

    return torch.stack(inputs_lst), torch.stack(targets_lst)
```

### 数据流

```
原始: [21106, 318, 281, 12064, ...]  (token IDs)
  ↓ custom_collate_fn
inputs:  [21106, 318, 281, ..., 50256, 50256]   ← pad 到等长
targets: [318, 281, 12064, ..., -100, -100]      ← 右移 + mask
  ↓ model forward
logits:  (batch, seq, vocab)  ← 每个 token 位置的预测
  ↓ flatten + cross_entropy
loss: 只在 targets != -100 的位置计算
```

## 模块 3: 训练实践

### 配置

```python
# 本书使用 355M 模型 (效果更好)
# 我们用 124M (硬件限制)

model = GPTModel(CFG_124M)
model.load_state_dict(torch.load("gpt2_pretrained.pth"))  # 预训练权重

optimizer = AdamW(lr=5e-5, weight_decay=0.1)
# 注意: 指令微调不替换输出层! 保留 vocab_size=50257
# 注意: 不冻结参数! 全量微调 (所有层都可训练)
```

### Loss 计算 (与 ch05 相同)

```python
logits = model(inputs)           # (batch, seq, vocab)
loss = F.cross_entropy(
    logits.flatten(0, 1),       # (batch*seq, vocab)
    targets.flatten()            # (batch*seq,)  -100 自动忽略
)
```

|-------|-----------|----------|------|
| 1 | 0.789 | 1.012 | 58s |
| 2 | 0.523 | 0.980 | 116s |

## ch06 vs ch07 关键差异

| 维度 | ch06 分类微调 | ch07 指令微调 |
|------|-------------|-------------|
| 输出层 | 替换为 num_classes | **保留** vocab_size |
| 冻结 | 冻结前 11 层 | **全量训练** |
| Loss 位置 | 最后 1 个 token | Response 部分 (所有 token) |
| Masking | 不需要 | **-100** 忽略 padding |
| 目标 | 分类 (spam/ham) | 生成文本回复 |
| collate | 固定长度 padding | **动态** padding (每 batch 不同) |

## 核心架构图

```
输入: "Below is an instruction... Identify animals: cats, dogs"

Token IDs: [21106, 318, ..., 379, 13]
     ↓ GPT-2 (全量微调, 所有 12 层)
Hidden states: (seq, 768)
     ↓ out_head: Linear(768 → 50257)  ← 不替换!
Logits: (seq, 50257)
     ↓ cross_entropy(logits, targets)
     ↓ targets 中 -100 的位置被忽略
Loss: 只在 Response 部分的 token 上计算

生成时:
  输入: "### Instruction:\nWhat is 2+2?\n\n### Response:\n"
  自回归生成: "The answer is 4."
```
