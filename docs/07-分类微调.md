## 二、核心知识点

### 2.1 分类微调概述

预训练模型只是基础——它们学会了语言的通用表示，但本身不能直接做分类。微调（finetuning）就是在预训练模型基础上进一步训练，使其适应特定任务。

**两种微调路线**：

| 路线 | 章节 | 任务 | 输出 |
|------|------|------|------|
| **分类微调** | 第 6 章 | 预测类别标签（spam / ham） | 离散类别 |
| **指令微调** | 第 7 章 | 回答问题、翻译、总结等 | 自由文本 |

**分类微调的两种实现方式**：

1. **替换输出头（Output Head Replacement）**：将原来的 `out_head`（输出 vocab_size 维的词概率）替换为新的分类层（输出 num_classes 维）。这是本章使用的方法。
2. **特征提取（Feature Extraction）**：完全冻结模型，只把它当特征提取器，将隐藏层输出送入独立的分类器。

**为什么分类微调值得学？**

不是所有场景都需要 ChatGPT 那样的通用对话能力。在实际业务中，大量需求只是"给一段文本，输出一个类别"——情感分析、垃圾邮件检测、主题分类、意图识别等。小模型 + 分类微调就能高效解决这些问题。

#### 2.2.2 处理类别不平衡

原始数据严重不平衡（ham 是 spam 的 6.5 倍）。本章采用**下采样（undersampling）**策略：

```python
def create_balanced_dataset(df):
    num_spam = df[df["Label"] == "spam"].shape[0]
    ham_subset = df[df["Label"] == "ham"].sample(num_spam, random_state=123)
    balanced_df = pd.concat([ham_subset, df[df["Label"] == "spam"]])
    return balanced_df
```

平衡后得到 1,494 条数据（747 ham + 747 spam）。

> 也可以用其他方法处理不平衡：上采样、类别权重加权、或使用 imbalanced-learn 库。

#### 2.2.3 数据集分割

将标签转为数值（ham=0, spam=1），按 70/10/20 分割：

```python
balanced_df["Label"] = balanced_df["Label"].map({"ham": 0, "spam": 1})
```

#### 2.2.4 分词与 Padding

使用 tiktoken（GPT-2 的 BPE tokenizer）进行分词，并将所有序列 **padding 到相同长度**：

```python
class SpamDataset(Dataset):
    def __init__(self, csv_file, tokenizer, max_length=None, pad_token_id=50256):
        self.data = pd.read_csv(csv_file)
        self.encoded_texts = [
            tokenizer.encode(text) for text in self.data["Text"]
        ]

        if max_length is None:
            self.max_length = self._longest_encoded_length()
        else:
            self.max_length = max_length
            self.encoded_texts = [
                encoded_text[:self.max_length]
                for encoded_text in self.encoded_texts
            ]

        # Padding: 用 <|endoftext|> token (50256) 填充短序列
        self.encoded_texts = [
            encoded_text + [pad_token_id] * (self.max_length - len(encoded_text))
            for encoded_text in self.encoded_texts
        ]
```

**关键细节**：
- 训练集的 `max_length` 取最长训练样本（120 tokens）
- 验证集和测试集使用**训练集的 max_length**，避免数据泄露
- Padding token 使用 `<|endoftext|>`（ID: 50256），这是 GPT-2 的默认 padding token

#### 2.2.5 创建 DataLoader

### 2.3 修改 GPT 架构

#### 2.3.1 加载预训练模型

```python
from gpt_download import download_and_load_gpt2

BASE_CONFIG = {
    "vocab_size": 50257,
    "context_length": 1024,
    "drop_rate": 0.0,
    "qkv_bias": True,
    "emb_dim": 768,
    "n_layers": 12,
    "n_heads": 12,
}

settings, params = download_and_load_gpt2(model_size="124M", models_dir="gpt2")
model = GPTModel(BASE_CONFIG)
load_weights_into_gpt(model, params)
```

加载后先验证模型是否正常工作（用 `generate_text_simple` 生成一段文本确认无误）。

#### 2.3.2 替换输出头

核心修改——只需一行代码：

```python
num_classes = 2
model.out_head = torch.nn.Linear(
    in_features=BASE_CONFIG["emb_dim"],  # 768
    out_features=num_classes              # 2 (spam / ham)
)
```

**为什么只改输出头？**

原始 `out_head` 将 768 维隐藏向量映射到 50,257 维（整个词表大小），用于预测下一个 token。分类任务不需要预测词，只需要映射到类别空间。新 `out_head` 从 768 维 → 2 维，参数量从 ~38M 降到 ~1.5K。

#### 2.3.3 为什么用最后一个 token？

分类时，我们从模型输出张量中取**最后一个 token 位置**的 logits：

```python
logits = model(input_batch)[:, -1, :]  # shape: (batch_size, num_classes)
```

原因在于 **causal attention mask**：

- 在 causal mask 下，第 1 个 token 只能看到自己
- 第 2 个 token 能看到第 1 个和第 2 个
- ...
- **最后一个 token 能看到所有 token**

因此，最后一个 token 的隐藏状态已经**聚合了整个输入序列的全局信息**，是做分类决策的最佳位置。

> 对照实验证明：用第一个 token 做分类，测试准确率从 95% 暴跌到 75%（Row 1 vs Row 2）。

### 2.4 冻结策略（Freezing Strategies）

#### 2.4.1 为什么要冻结

对于小数据集（~1,000 条），训练整个模型（124M 参数）容易过拟合，而且训练时间长。冻结底层参数可以：
- 减少可训练参数数量
- 加快训练速度
- 降低过拟合风险
- 节省显存

#### 2.4.2 冻结实现

```python
# 第一步：冻结所有参数
for param in model.parameters():
    param.requires_grad = False

# 第二步：替换输出头（新层默认 requires_grad=True）
model.out_head = torch.nn.Linear(BASE_CONFIG["emb_dim"], num_classes)

# 第三步：解冻最后一个 Transformer block + final LayerNorm
for param in model.trf_blocks[-1].parameters():
    param.requires_grad = True
for param in model.final_norm.parameters():
    param.requires_grad = True
```

#### 2.4.3 不同策略的实验对比

| 策略 | 可训练层 | 测试准确率 | 训练时间 |
|------|----------|-----------|---------|
| 只训练输出头 | `last_layer` | 72.00% | 0.25 min |
| **训练最后 block + 输出头** | `last_block` | **95.00%** | **0.28 min** |
| 训练最后两个 block | `last_two_blocks` | 98.33% | 0.33 min |
| 训练所有层 | `all` | 96.67% | 0.69 min |

**结论**：
- 训练最后一个 Transformer block 是**性价比最高**的选择（95% 准确率，仅需 0.28 min）
- 训练最后两个 block 能提升到 98.33%，但收益递减
- 训练所有层反而不如训练两个 block（过拟合），且慢 2.5 倍
- 只训练最后一层（不包含整个 block）效果很差（72%）

> 底层学到的是通用语言特征（语法、词汇关系），具有迁移性，不需要重新训练。顶层学到的是任务特定特征，需要微调。

### 2.5 训练与评估

#### 2.5.1 损失函数

分类任务使用 **cross-entropy loss**：

```python
def calc_loss_batch(input_batch, target_batch, model, device):
    input_batch, target_batch = input_batch.to(device), target_batch.to(device)
    logits = model(input_batch)[:, -1, :]  # 取最后一个 token 的输出
    loss = torch.nn.functional.cross_entropy(logits, target_batch)
    return loss
```

与预训练不同：
- 预训练：loss 在**所有 token 位置**计算
- 分类：loss 只在**最后一个 token 位置**计算

#### 2.5.2 准确率计算

```python
def calc_accuracy_loader(data_loader, model, device, num_batches=None):
    model.eval()
    correct_predictions, num_examples = 0, 0

    for i, (input_batch, target_batch) in enumerate(data_loader):
        if i >= num_batches:
            break
        input_batch, target_batch = input_batch.to(device), target_batch.to(device)
        with torch.no_grad():
            logits = model(input_batch)[:, -1, :]
        predicted_labels = torch.argmax(logits, dim=-1)
        num_examples += predicted_labels.shape[0]
        correct_predictions += (predicted_labels == target_batch).sum().item()

    return correct_predictions / num_examples
```

#### 2.5.3 训练循环

```python
optimizer = torch.optim.AdamW(model.parameters(), lr=5e-5, weight_decay=0.1)

num_epochs = 5
train_losses, val_losses, train_accs, val_accs, examples_seen = train_classifier_simple(
    model, train_loader, val_loader, optimizer, device,
    num_epochs=num_epochs, eval_freq=50, eval_iter=5,
)
```

关键超参数：
- **学习率**：5e-5（比预训练低一个数量级）
- **weight decay**：0.1（正则化，防止过拟合）
- **epoch 数**：5（小数据集不需要太多轮）

### 2.6 实际使用

#### 2.6.1 在新文本上预测

```python
def classify_text(text, model, tokenizer, device, max_length):
    model.eval()

    # 1. 分词
    encoded = tokenizer.encode(text)
    # 2. 截断
    encoded_tensor = torch.tensor(
        encoded[:max_length] + [50256] * (max_length - len(encoded)),
        dtype=torch.long
    ).unsqueeze(0).to(device)

    # 3. 前向推理
    with torch.no_grad():
        logits = model(encoded_tensor)[:, -1, :]

    # 4. 取 argmax 得到预测类别
    predicted_label = torch.argmax(logits, dim=-1).item()

    return "Spam" if predicted_label == 1 else "Not Spam"
```

#### 2.6.2 测试示例

```python
text_1 = "You have won a $1,000 gift card. Click here to claim!"
print(classify_text(text_1, model, tokenizer, device, train_dataset.max_length))
# 输出: Spam

text_2 = "Hey, just wanted to check if we're still on for dinner tonight?"
print(classify_text(text_2, model, tokenizer, device, train_dataset.max_length))
# 输出: Not Spam
```

#### 2.6.3 测试集全面评估

```python
train_accuracy = calc_accuracy_loader(train_loader, model, device)
val_accuracy = calc_accuracy_loader(val_loader, model, device)

```

## 四、常见问题

**Q: 为什么用最后一个 token 而不是第一个？**

A: 因为 causal attention mask。第一个 token 只能看到自己，信息量最少。最后一个 token 通过注意力机制已经看到了前面所有 token 的信息，聚合了全局上下文，所以最适合做分类决策。实验也证实了这一点：用第一个 token 准确率只有 75%，用最后一个 token 达到 95%。

**Q: 冻结多少层合适？**

A: 取决于数据集大小和任务复杂度。一般原则：
- 数据少（几百到几千条）→ 只训练最后一个 block
- 数据中等 → 训练最后 2-3 个 block
- 数据充足 → 可以训练所有层（但注意过拟合）
- 不确定时，从少到多实验，观察验证集准确率

**Q: 数据集很小（几百条）够用吗？**

A: 对于简单分类任务（如二分类垃圾邮件），几百条数据配合预训练模型通常够用。预训练模型已经学会了语言表示，微调只需要教会它"如何将这些表示映射到类别"。但数据越少，越需要谨慎：使用冻结策略、weight decay、早停等防止过拟合。

**Q: 分类微调和特征提取有什么区别？**

A:
- **分类微调**：替换输出头，解冻部分层，端到端训练。模型权重会更新，能更好地适应任务。
- **特征提取**：完全冻结模型，把隐藏层输出当特征，送入独立分类器（如 SVM、Logistic Regression）。更简单、更快，但灵活度较低。
- 本章使用分类微调，因为它在多数情况下效果更好。

**Q: 如何处理类别不平衡？**

A: 常用方法：
1. **下采样（undersampling）**：减少多数类样本（本章使用）
2. **上采样（oversampling）**：复制少数类样本
3. **类别权重**：在 loss 中给少数类更高权重（`cross_entropy` 的 `weight` 参数）
4. **Focal Loss**：自动降低易分类样本的权重

**Q: 为什么用 GPT 而不是 BERT 做分类？**

A: BERT 是双向模型，天然适合分类（用 `[CLS]` token）。GPT 是单向模型，但通过 causal mask 最后一个 token 也能聚合全局信息。GPT-2 在这个垃圾邮件任务上达到了 95% 的准确率。BERT 系列在更大规模数据集（如 IMDb 50k）上可能更好，但 GPT 的分类能力也不容小觑。详见 `ch06/03_bonus_imdb-classification/` 中的对比实验。

<summary><b>Q21: SpamDataset 中 padding 的作用是什么？为什么要用训练集的 max_length？</b></summary>

**A:**
- **Padding**：将不同长度的序列补齐到相同长度，使它们能组成 batch 送入模型
- 使用 token ID 50256（`<|endoftext|>`）作为 padding token
- 验证集和测试集必须使用**训练集的 max_length**（120），而非自己的最大长度
- 原因：避免数据泄露（测试数据不应影响模型配置），且保持与训练时一致的输入维度

</details>

---

<details>
<summary><b>Q22: 分类微调的训练超参数和预训练有什么不同？</b></summary>

**A:**

| 参数 | 预训练 | 分类微调 |
|------|--------|---------|
| 学习率 | 5e-4 | 5e-5（低 10 倍） |
| Loss 计算位置 | 所有 token 位置 | 仅最后一个 token |
| 训练数据量 | 大规模文本 | 任务特定标签数据 |
| 训练目标 | 预测下一个 token | 预测类别标签 |

微调用更小的学习率，避免破坏预训练学到的特征。

</details>
