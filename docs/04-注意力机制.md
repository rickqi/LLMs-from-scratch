## 二、核心知识点

### 2.1 简单注意力机制（Simple Attention）

**为什么需要 attention？**

Transformer 之前的模型（如 RNN）按顺序处理 token，难以捕捉远距离依赖。Attention 机制让每个 token 都能直接"看到"序列中的所有其他 token，根据相关性分配不同的权重。

**核心直觉**：对于序列中的某个 token（称为 query），计算它与所有 token（包括自身）的相似度，用 softmax 归一化后作为权重，对所有 token 的向量做加权求和，得到一个融合了上下文信息的新向量（context vector）。

**步骤拆解**：

```python
import torch

# 输入: 6 个 token，每个 3 维 embedding
inputs = torch.tensor(
    [[0.43, 0.15, 0.89],  # x1 (Your)
     [0.55, 0.87, 0.66],  # x2 (journey)
     [0.57, 0.85, 0.64],  # x3 (starts)
     [0.22, 0.58, 0.33],  # x4 (with)
     [0.77, 0.25, 0.10],  # x5 (one)
     [0.05, 0.80, 0.55]]  # x6 (step)
)

# 步骤 1: 计算 attention scores（点积）
# 以 x2 作为 query，与所有 token 计算点积
query = inputs[1]                            # x2: (3,)
attn_scores_2 = torch.empty(inputs.shape[0])
for i, x_i in enumerate(inputs):
    attn_scores_2[i] = torch.dot(x_i, query) # 每个 token 与 query 的相似度

# 步骤 2: softmax 归一化 → attention weights
attn_weights_2 = torch.softmax(attn_scores_2, dim=0)

# 步骤 3: 加权求和 → context vector
query_2_context = torch.zeros(query.shape)
for i, x_i in enumerate(inputs):
    query_2_context += attn_weights_2[i] * x_i
```

**用矩阵运算替代循环**：

```python
# 所有 query 一次性计算: attn_scores = inputs @ inputs^T
attn_scores = inputs @ inputs.T              # (6, 6)
attn_weights = torch.softmax(attn_scores, dim=-1)  # (6, 6)

# 所有权重一次加权求和
all_context_vecs = attn_weights @ inputs     # (6, 3)
```

**关键洞察**：点积衡量两个向量的相似度。softmax 将相似度转化为概率分布（权重和为 1）。加权求和让每个 token 融合全局信息。

### 2.2 自注意力机制（Self-Attention）

简单版 attention 直接用输入向量做点积，但真实模型需要**可训练的参数**来学习"什么是相关的"。Self-attention 引入三个可学习的投影矩阵 Wq、Wk、Wv，将输入投影到三个不同的空间：

| 概念 | 含义 | 类比 |
|------|------|------|
| **Q**（Query） | "我在找什么？" | 搜索查询 |
| **K**（Key） | "我有什么特征？" | 搜索关键词 |
| **V**（Value） | "我的实际内容" | 搜索结果 |

**完整公式**：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

**代码实现**：

```python
import torch
import torch.nn as nn

class SelfAttention_v1(nn.Module):
    def __init__(self, d_in, d_out):
        super().__init__()
        self.W_query = nn.Parameter(torch.rand(d_in, d_out))
        self.W_key   = nn.Parameter(torch.rand(d_in, d_out))
        self.W_value = nn.Parameter(torch.rand(d_in, d_out))

    def forward(self, x):
        keys    = x @ self.W_key
        queries = x @ self.W_query
        values  = x @ self.W_value
        attn_scores = queries @ keys.T
        attn_weights = torch.softmax(attn_scores / keys.shape[-1]**0.5, dim=-1)
        context_vec = attn_weights @ values
        return context_vec
```

**更实用的版本（用 `nn.Linear`）**：

```python
class SelfAttention_v2(nn.Module):
    def __init__(self, d_in, d_out, qkv_bias=False):
        super().__init__()
        self.W_query = nn.Linear(d_in, d_out, bias=qkv_bias)
        self.W_key   = nn.Linear(d_in, d_out, bias=qkv_bias)
        self.W_value = nn.Linear(d_in, d_out, bias=qkv_bias)

    def forward(self, x):
        keys    = self.W_key(x)
        queries = self.W_query(x)
        values  = self.W_value(x)
        attn_scores = queries @ keys.T
        attn_weights = torch.softmax(attn_scores / keys.shape[-1]**0.5, dim=-1)
        context_vec = attn_weights @ values
        return context_vec
```

**为什么除以 $\sqrt{d_k}$？** 见 [常见问题 Q1](#四常见问题)。

### 2.3 因果注意力（Causal Attention）

Self-attention 让每个 token 都能看到所有 token——但在 LLM 中，生成第 t 个 token 时**不能偷看**第 t+1 个及之后的 token（否则等于作弊）。

**解决方案：因果掩码（Causal Mask）**

用一个上三角矩阵将"未来位置"设为 `-inf`，softmax 后这些位置变为 0：

```
[[  0, -inf, -inf, -inf, -inf, -inf],
 [  0,    0, -inf, -inf, -inf, -inf],
 [  0,    0,    0, -inf, -inf, -inf],
 [  0,    0,    0,    0, -inf, -inf],
 [  0,    0,    0,    0,    0, -inf],
 [  0,    0,    0,    0,    0,    0 ]]
```

**代码实现**：

```python
class CausalAttention(nn.Module):
    def __init__(self, d_in, d_out, context_length, dropout=0.0, qkv_bias=False):
        super().__init__()
        self.d_out = d_out
        self.W_query = nn.Linear(d_in, d_out, bias=qkv_bias)
        self.W_key   = nn.Linear(d_in, d_out, bias=qkv_bias)
        self.W_value = nn.Linear(d_in, d_out, bias=qkv_bias)
        self.dropout = nn.Dropout(dropout)

        # 关键：用 register_buffer 注册 mask
        mask = torch.triu(torch.ones(context_length, context_length), diagonal=1)
        self.register_buffer("mask", torch.BoolTensor(mask.bool()))

    def forward(self, x):
        b, num_tokens, d_in = x.shape
        keys    = self.W_key(x)
        queries = self.W_query(x)
        values  = self.W_value(x)

        attn_scores = queries @ keys.transpose(1, 2)
        # 将上三角（未来位置）替换为 -inf
        attn_scores.masked_fill_(self.mask[:num_tokens, :num_tokens], -torch.inf)
        attn_weights = torch.softmax(attn_scores / self.d_out**0.5, dim=-1)
        attn_weights = self.dropout(attn_weights)

        context_vec = attn_weights @ values
        return context_vec
```

**`register_buffer` vs `requires_grad=False`**

这是第 3 章中容易混淆的一个点（详见 [PyTorch Buffers 视频](https://www.youtube.com/watch?v=PetlIokI9Ao)）：

| 特性 | `register_buffer` | `self.x = tensor; x.requires_grad=False` |
|------|-------------------|-------------------------------------------|
| 会被 `model.to(device)` 自动转移 | ✅ 是 | ❌ 否——留在 CPU 上 |
| 会出现在 `state_dict()` 中 | ✅ 是 | ❌ 否 |
| 参与梯度计算 | ❌ 否 | ❌ 否 |
| 保存/加载模型时自动处理 | ✅ 是 | ❌ 否 |

**核心原因**：当 `model.to("cuda")` 时，`register_buffer` 注册的张量会跟着一起转移到 GPU，而普通的 attribute 不会。如果 mask 留在 CPU 上，而模型权重在 GPU 上，前向传播就会报错：

```
RuntimeError: Expected all tensors to be on the same device, but found at least two devices, cuda:0 and cpu!
```

### 2.4 多头注意力（Multi-Head Attention）

**直觉**：单头注意力只能学习一种"关注模式"。多头注意力让模型同时从多个不同角度关注输入——每个头学习不同的关系模式。

**实现方式**：

```python
class MultiHeadAttention(nn.Module):
    def __init__(self, d_in, d_out, context_length, dropout, num_heads, qkv_bias=False):
        super().__init__()
        assert d_out % num_heads == 0, "d_out must be divisible by num_heads"

        self.d_out = d_out
        self.num_heads = num_heads
        self.head_dim = d_out // num_heads  # 每个头的维度

        self.W_query = nn.Linear(d_in, d_out, bias=qkv_bias)
        self.W_key   = nn.Linear(d_in, d_out, bias=qkv_bias)
        self.W_value = nn.Linear(d_in, d_out, bias=qkv_bias)
        self.out_proj = nn.Linear(d_out, d_out)  # 输出投影
        self.dropout = nn.Dropout(dropout)

        mask = torch.triu(torch.ones(context_length, context_length), diagonal=1)
        self.register_buffer("mask", torch.BoolTensor(mask.bool()))

    def forward(self, x):
        b, num_tokens, d_in = x.shape

        # 1. Q/K/V 投影
        keys    = self.W_key(x)       # (b, num_tokens, d_out)
        queries = self.W_query(x)
        values  = self.W_value(x)

        # 2. 拆分为多头: (b, num_tokens, d_out) → (b, num_tokens, num_heads, head_dim) → (b, num_heads, num_tokens, head_dim)
        keys    = keys.view(b, num_tokens, self.num_heads, self.head_dim).transpose(1, 2)
        queries = queries.view(b, num_tokens, self.num_heads, self.head_dim).transpose(1, 2)
        values  = values.view(b, num_tokens, self.num_heads, self.head_dim).transpose(1, 2)

        # 3. 计算注意力
        attn_scores = queries @ keys.transpose(2, 3)  # (b, num_heads, num_tokens, num_tokens)
        attn_scores.masked_fill_(self.mask[:num_tokens, :num_tokens], -torch.inf)
        attn_weights = torch.softmax(attn_scores / self.head_dim**0.5, dim=-1)
        attn_weights = self.dropout(attn_weights)

        # 4. 加权求和
        context_vec = (attn_weights @ values).transpose(1, 2)  # (b, num_heads, num_tokens, head_dim) → (b, num_tokens, num_heads, head_dim)

        # 5. 合并多头 + 输出投影
        context_vec = context_vec.contiguous().view(b, num_tokens, self.d_out)  # (b, num_tokens, d_out)
        context_vec = self.out_proj(context_vec)
        return context_vec
```

**张量形状变化一览**（以 GPT-2 小模型为例）：

| 步骤 | 形状 | 示例 |
|------|------|------|
| 输入 x | `(batch, num_tokens, d_in)` | `(2, 6, 768)` |
| Q/K/V 投影 | `(batch, num_tokens, d_out)` | `(2, 6, 768)` |
| 拆分多头 (view + transpose) | `(batch, num_heads, num_tokens, head_dim)` | `(2, 12, 6, 64)` |
| 注意力分数 Q@K^T | `(batch, num_heads, num_tokens, num_tokens)` | `(2, 12, 6, 6)` |
| 加权求和 attn@V | `(batch, num_heads, num_tokens, head_dim)` | `(2, 12, 6, 64)` |
| 合并多头 (transpose + view) | `(batch, num_tokens, d_out)` | `(2, 6, 768)` |
| 输出投影 | `(batch, num_tokens, d_out)` | `(2, 6, 768)` |

> GPT-2 配置：`d_out = 768`, `num_heads = 12`, `head_dim = 768 / 12 = 64`

### 2.5 注意力机制总结

从简化版到完整版，逐步增加复杂度：

| 版本 | 新增内容 | 关键特点 |
|------|---------|---------|
| **Simple Attention** | 点积 + softmax | 无可训练参数，直接用输入做点积 |
| **Self-Attention** | Q/K/V 投影 + 缩放 | 引入可学习参数，除以 $\sqrt{d_k}$ |
| **Causal Attention** | 因果掩码 + dropout | 防止"偷看"未来 token |
| **Multi-Head Attention** | 多头并行 + 输出投影 | 多角度关注，`out_proj` 融合信息 |

**演进路径**：

```
输入 tokens
    ↓
简单点积 → softmax → 加权求和           (Simple Attention)
    ↓ + 可学习 Q/K/V 投影 + 缩放
缩放点积注意力                            (Self-Attention)
    ↓ + 上三角掩码 (-inf)
因果注意力                                (Causal Attention)
    ↓ + 拆分多头 + 输出投影
多头注意力                                (Multi-Head Attention)
```

## 四、常见问题

**Q: 为什么注意力分数要除以 $\sqrt{d_k}$？**

A: 当 `d_k`（每个头的维度）较大时，点积 $Q \cdot K^T$ 的值方差会增大。经过 softmax 后，大的值趋近 1，小的趋近 0——导致梯度几乎为零（softmax 饱和）。除以 $\sqrt{d_k}$ 将方差控制在 1，保持 softmax 输出"软"一些，梯度更稳定。例如 GPT-2 的 `head_dim=64`，除以 8（即 $\sqrt{64}$）。

**Q: attention 和 self-attention 有什么区别？**

A: Self-attention 是 attention 的特例——Q、K、V 都来自同一个输入序列。而一般的 attention 中，Q 可能来自一个序列，K 和 V 来自另一个序列（称为 cross-attention，如翻译模型中 decoder 用 encoder 的输出作为 K/V）。LLM 中几乎全部使用 self-attention。

**Q: 为什么用 `register_buffer` 而不是 `requires_grad=False`？**

A: 核心区别在设备迁移。`model.to("cuda")` 会自动转移 buffer，但不会转移普通 attribute。如果 mask 是普通 attribute 且留在 CPU，而权重在 GPU，就会报 `RuntimeError`。此外，buffer 会自动包含在 `state_dict()` 中，保存/加载模型时无需额外处理。

**Q: 多头注意力中 `num_heads` 和 `head_dim` 是什么关系？**

A: `d_out = num_heads × head_dim`。例如 GPT-2 的 `d_out=768`，`num_heads=12`，则 `head_dim=64`。每个头独立计算注意力，最后拼接。增加头数不改变总计算量，但让模型能学习多种不同的关注模式。

**Q: 多头注意力为什么需要一个 `out_proj`（输出投影层）？**

A: 多个头的输出拼接后，维度是 `d_out`，但信息来自多个独立子空间。`out_proj`（一个 `nn.Linear`）的作用是在所有头的输出之间做信息融合，让模型学习如何组合不同头学到的表示。没有它，各头信息就是简单拼接，缺乏跨头交互。

**Q: `torch.triu` 生成的 mask 为什么用 `bool` 类型？**

A: `masked_fill_` 接受 bool 类型的 mask——`True` 的位置会被替换为指定值（`-inf`）。使用 bool 比 float 更清晰，也避免了浮点比较的精度问题。`register_buffer("mask", mask.bool())` 将 bool tensor 注册为 buffer。

<summary><b>Q7: 自注意力的计算公式是什么？写出计算步骤。</b></summary>

**A:**
$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

步骤：
1. 从输入 x 投影出 Q、K、V：Q=x·Wq, K=x·Wk, V=x·Wv
2. 计算注意力分数：scores = Q @ K^T
3. 缩放：scores / sqrt(d_k)——防止点积过大导致 softmax 梯度消失
4. 应用因果掩码（causal mask）：将未来位置设为 -inf
5. Softmax 归一化：attn_weights = softmax(scores)
6. 加权求和：output = attn_weights @ V

<details>
<summary><b>Q10: 为什么注意力分数要除以 sqrt(d_k)？</b></summary>

**A:** 当 d_k（每个头的维度）较大时，点积 Q·K^T 的值方差会增大。经过 softmax 后，大的值会趋近于 1，小的趋近于 0——导致梯度几乎为零（softmax 饱和）。除以 sqrt(d_k) 将方差控制为 1，保持 softmax 输出"软"一些，梯度更稳定。

</details>

## 八、前沿延伸：注意力机制的新发现与新问题

> 以下内容整合自 [How LLMs Actually Work](https://www.0xkato.xyz/how-llms-actually-work/)（0xkato, 2025），作为注意力机制的进阶阅读。

### 8.1 Induction Heads：In-Context Learning 的机制解释

**这是可解释性研究里最有意思的发现之一**（Anthropic, 2022）。

某些注意力头在训练中自发学会了识别 prompt 中 "A B … A" 形态的模式并预测 B 接着出现——第二次看到 A 时，induction head 回看 A 上次出现的位置，看它后面跟了什么，然后照抄。

这就是 **in-context learning**（模型从你的 prompt 里现学模式并续写的能力）目前已知最清晰的机制之一。

> 一个典型的 12 层 GPT-2 模型有 144 个注意力头，其中相当比例发展出了 induction 行为，没有人告诉它们该这么做——这是训练中自发涌现的。

**与本书的关系**：你实现的 `MultiHeadAttention` 就是这些 head 的载体。induction heads 不是特殊的架构组件，而是 12 个头上学到的权重组合所产生的新行为。

### 8.2 "Lost in the Middle"：长上下文的隐藏缺陷

即便有了因果掩码和高效的注意力实现，现代 LLM 仍有被文献记录的 **"lost in the middle"** 问题（Liu et al., 2023）：

> 长 prompt 开头和结尾的信息被利用得比中间的可靠。模型并没有均匀地利用 prompt 的每个部分。

这解释了为什么以下 prompt 工程技巧真的有效：
- **重要上下文放前面**（指令前置）
- **关键信息在结尾重复一遍**
- **结构化分段**，让模型更容易定位

### 8.3 注意力的代价：长 prompt 为什么贵

完整注意力中每个 token 要和所有可见 token 比较——prompt 长度翻倍，计算量大约翻四倍（O(n²)）。这也是 FlashAttention、稀疏注意力、线性注意力、MLA、SWA 等效率研究的动机。

> 本书 `ch04/` 补充目录下覆盖了多种注意力替代方案：GQA、MLA、SWA、DeltaNet、DSA、KV Sharing。
