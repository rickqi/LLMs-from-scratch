### 2.2 训练循环（Training Loop）

训练循环是标准的 PyTorch 深度学习流程：

```
for epoch in range(num_epochs):
    for batch in train_loader:
        optimizer.zero_grad()   # 1. 清零梯度
        loss = forward(batch)   # 2. 前向传播 + 计算损失
        loss.backward()         # 3. 反向传播（计算梯度）
        optimizer.step()        # 4. 更新权重
```

**完整训练函数**：

```python
def train_model_simple(model, train_loader, val_loader, optimizer, device,
                       num_epochs, eval_freq, eval_iter, start_context, tokenizer):
    train_losses, val_losses, track_tokens_seen = [], [], []
    tokens_seen, global_step = 0, -1

    for epoch in range(num_epochs):
        model.train()
        for input_batch, target_batch in train_loader:
            optimizer.zero_grad()
            loss = calc_loss_batch(input_batch, target_batch, model, device)
            loss.backward()
            optimizer.step()
            tokens_seen += input_batch.numel()
            global_step += 1

            if global_step % eval_freq == 0:
                train_loss, val_loss = evaluate_model(
                    model, train_loader, val_loader, device, eval_iter)
                train_losses.append(train_loss)
                val_losses.append(val_loss)
                track_tokens_seen.append(tokens_seen)
                print(f"Ep {epoch+1} (Step {global_step:06d}): "
                      f"Train loss {train_loss:.3f}, Val loss {val_loss:.3f}")

        generate_and_print_sample(model, tokenizer, device, start_context)

    return train_losses, val_losses, track_tokens_seen
```

**关键细节**：
- `model.train()` 启用 Dropout；`model.eval()` 禁用 Dropout
- 每隔 `eval_freq` 步评估一次训练和验证损失
- 每个 epoch 结束后生成一段示例文本，观察模型输出质量
- 使用 `torch.no_grad()` 在评估时禁用梯度计算，节省内存

**超参数**：


### 2.3 交叉熵损失

LLM 预训练的损失函数是**交叉熵损失（Cross-Entropy Loss）**，等价于**负平均对数似然（Negative Average Log-Likelihood）**。

**直观理解**：对每个位置，模型输出一个 vocab_size 维的 logits 向量，表示对下一个 token 的预测。损失衡量预测分布与真实 token 之间的差距。

**计算步骤分解**：
1. Logits → Softmax → 概率分布
2. 取目标 token 对应的概率
3. 取对数 → 取负 → 取平均

**PyTorch 中的实现**：

```python
import torch.nn.functional as F

loss = F.cross_entropy(logits.flatten(0, 1), targets.flatten())
```

**为什么要 flatten？**

Logits 的形状是 `(batch_size, seq_len, vocab_size)`，即 `(2, 256, 50257)`。
Targets 的形状是 `(batch_size, seq_len)`，即 `(2, 256)`。

`F.cross_entropy` 期望输入为 2D `(N, C)` 和 1D `(N,)`，所以需要将 batch 和 seq_len 维度合并：

```python
logits.flatten(0, 1)   # (batch*seq_len, vocab_size) = (512, 50257)
targets.flatten()       # (batch*seq_len,) = (512,)
```

**损失函数辅助代码**：

```python
def calc_loss_batch(input_batch, target_batch, model, device):
    input_batch, target_batch = input_batch.to(device), target_batch.to(device)
    logits = model(input_batch)
    loss = torch.nn.functional.cross_entropy(
        logits.flatten(0, 1), target_batch.flatten()
    )
    return loss


def calc_loss_loader(data_loader, model, device, num_batches=None):
    total_loss = 0.
    if num_batches is None:
        num_batches = len(data_loader)
    else:
        num_batches = min(num_batches, len(data_loader))
    for i, (input_batch, target_batch) in enumerate(data_loader):
        if i < num_batches:
            loss = calc_loss_batch(input_batch, target_batch, model, device)
            total_loss += loss.item()
        else:
            break
    return total_loss / num_batches
```

**未训练模型的初始损失**约为 `10.98`，接近 `ln(50257) ≈ 10.82`，意味着模型对每个 token 的预测是完全随机的。

### 2.4 文本生成（Text Generation）

#### 贪婪解码（Greedy Decoding）

最简单的生成策略：每一步都选概率最高的 token。

**算法步骤**：
1. 给定初始提示词 → tokenize → 得到 `idx` `(1, prompt_len)`
2. 循环 `max_new_tokens` 次：
   a. 截取最后 `context_length` 个 token 作为输入：`idx_cond = idx[:, -context_size:]`
   b. 前向传播：`logits = model(idx_cond)`
   c. 取最后一个位置的 logits：`logits = logits[:, -1, :]`
   d. 取 argmax：`idx_next = torch.argmax(logits, dim=-1, keepdim=True)`
   e. 拼接到序列末尾：`idx = torch.cat((idx, idx_next), dim=1)`
3. 解码 `idx` → 文本

**基础生成函数**：

```python
def generate_text_simple(model, idx, max_new_tokens, context_size):
    for _ in range(max_new_tokens):
        idx_cond = idx[:, -context_size:]
        with torch.no_grad():
            logits = model(idx_cond)
        logits = logits[:, -1, :]
        idx_next = torch.argmax(logits, dim=-1, keepdim=True)
        idx = torch.cat((idx, idx_next), dim=1)
    return idx
```

**辅助函数**（文本 ↔ token IDs 互转）：

```python
def text_to_token_ids(text, tokenizer):
    encoded = tokenizer.encode(text)
    return torch.tensor(encoded).unsqueeze(0)  # 添加 batch 维度


def token_ids_to_text(token_ids, tokenizer):
    flat = token_ids.squeeze(0)  # 去掉 batch 维度
    return tokenizer.decode(flat.tolist())
```

#### 增强生成：Temperature 和 Top-k 采样

贪婪解码的缺点：每次输出都一样，缺乏多样性。解决方案：

**Temperature Scaling**：控制随机性
- `temperature > 1` → 更随机（更均匀的分布）
- `temperature < 1` → 更确定（更尖锐的分布）
- `temperature = 0` → 退化为贪婪解码

```python
if temperature > 0.0:
    logits = logits / temperature
    probs = torch.softmax(logits, dim=-1)
    idx_next = torch.multinomial(probs, num_samples=1)
```

**Top-k Sampling**：只保留概率最高的 k 个 token

```python
if top_k is not None:
    top_logits, _ = torch.topk(logits, top_k)
    min_val = top_logits[:, -1]
    logits = torch.where(
        logits < min_val,
        torch.tensor(float("-inf")).to(logits.device),
        logits
    )
```

**完整 generate 函数**：

```python
def generate(model, idx, max_new_tokens, context_size,
             temperature=0.0, top_k=None, eos_id=None):
    for _ in range(max_new_tokens):
        idx_cond = idx[:, -context_size:]
        with torch.no_grad():
            logits = model(idx_cond)
        logits = logits[:, -1, :]

        if top_k is not None:
            top_logits, _ = torch.topk(logits, top_k)
            min_val = top_logits[:, -1]
            logits = torch.where(
                logits < min_val,
                torch.tensor(float("-inf")).to(logits.device),
                logits
            )

        if temperature > 0.0:
            logits = logits / temperature
            probs = torch.softmax(logits, dim=-1)
            idx_next = torch.multinomial(probs, num_samples=1)
        else:
            idx_next = torch.argmax(logits, dim=-1, keepdim=True)

        if idx_next == eos_id:
            break
        idx = torch.cat((idx, idx_next), dim=1)

    return idx
```

### 2.5 加载预训练权重（Loading Pretrained Weights）

在少量文本上训练 5 分钟的模型输出质量很差。幸运的是，OpenAI 公开了 GPT-2 的预训练权重，我们可以直接加载。

#### 下载 GPT-2 权重

使用 `gpt_download.py` 从 OpenAI 下载权重文件（TensorFlow 格式）：

```python
from gpt_download import download_and_load_gpt2

settings, params = download_and_load_gpt2(
    model_size="124M", models_dir="gpt2"
)
```

权重文件包含：`checkpoint`、`encoder.json`、`hparams.json`、`model.ckpt.*`、`vocab.bpe`。

#### 映射 OpenAI 权重到我们的模型

OpenAI 的权重名称与我们的 `GPTModel` 结构不同，需要手动映射：

```python
def assign(left, right):
    if left.shape != right.shape:
        raise ValueError(f"Shape mismatch. Left: {left.shape}, Right: {right.shape}")
    return torch.nn.Parameter(torch.tensor(right))


def load_weights_into_gpt(gpt, params):
    gpt.pos_emb.weight = assign(gpt.pos_emb.weight, params["wpe"])
    gpt.tok_emb.weight = assign(gpt.tok_emb.weight, params["wte"])

    for b in range(len(params["blocks"])):
        # QKV 权重：OpenAI 合并存储，需要拆分
        q_w, k_w, v_w = np.split(
            (params["blocks"][b]["attn"]["c_attn"])["w"], 3, axis=-1)
        gpt.trf_blocks[b].att.W_query.weight = assign(
            gpt.trf_blocks[b].att.W_query.weight, q_w.T)
        gpt.trf_blocks[b].att.W_key.weight = assign(
            gpt.trf_blocks[b].att.W_key.weight, k_w.T)
        gpt.trf_blocks[b].att.W_value.weight = assign(
            gpt.trf_blocks[b].att.W_value.weight, v_w.T)
        # ... 其余权重映射类似
```

**关键映射关系**：

| OpenAI 名称 | 我们的模型名称 |
|------------|--------------|
| `wpe` | `pos_emb.weight` |
| `wte` | `tok_emb.weight` / `out_head.weight` |
| `blocks[b].attn.c_attn.w` | `W_query.weight` + `W_key.weight` + `W_value.weight` |
| `blocks[b].attn.c_proj.w` | `out_proj.weight` |
| `blocks[b].mlp.c_fc.w` | `ff.layers[0].weight` |
| `blocks[b].ln_1.g` | `norm1.scale` |

> 注意：`out_head.weight` 与 `tok_emb.weight` 共享（weight tying），这是 GPT-2 的设计。

### 2.6 训练监控与困惑度

#### 损失曲线

训练过程中跟踪两个指标：
- **Training Loss**：在训练集上计算的损失
- **Validation Loss**：在验证集上计算的损失

```python
def evaluate_model(model, train_loader, val_loader, device, eval_iter):
    model.eval()
    with torch.no_grad():
        train_loss = calc_loss_loader(train_loader, model, device, num_batches=eval_iter)
        val_loss = calc_loss_loader(val_loader, model, device, num_batches=eval_iter)
    model.train()
    return train_loss, val_loss
```

绘制损失曲线：

```python
def plot_losses(epochs_seen, tokens_seen, train_losses, val_losses):
    fig, ax1 = plt.subplots()
    ax1.plot(epochs_seen, train_losses, label="Training loss")
    ax1.plot(epochs_seen, val_losses, linestyle="-.", label="Validation loss")
    ax1.set_xlabel("Epochs")
    ax1.set_ylabel("Loss")
    ax1.legend(loc="upper right")

    ax2 = ax1.twiny()
    ax2.plot(tokens_seen, train_losses, alpha=0)
    ax2.set_xlabel("Tokens seen")
    fig.tight_layout()
```

#### 困惑度（Perplexity）

困惑度是交叉熵损失的另一种表示方式：

```
Perplexity = exp(loss) = torch.exp(loss)
```

**含义**：模型在预测下一个 token 时，平均在多少个候选词之间犹豫。

| Loss | Perplexity | 含义 |
|------|-----------|------|
| 10.98 | ~58,000 | 未训练，几乎完全随机（≈vocab_size） |
| 3.0 | ~20 | 较差，在 20 个词间犹豫 |
| 1.0 | ~2.7 | 较好 |
| 0.5 | ~1.6 | 很好，几乎确定下一个词 |

**典型训练趋势**：
- 初始 loss ≈ `ln(50257) ≈ 10.82`
- 训练后 loss 降到 `1.0` 以下就算不错
- 两者同时下降 → 模型在学习（正常）
- 训练 loss 下降，验证 loss 上升 → 过拟合

## 四、常见问题

**Q: 为什么训练这么慢？**
A: 本书使用 CPU 训练，且没有做任何性能优化。如有 GPU，可设置 `device = torch.device("cuda")`。还可以参考 `ch05/10_llm-training-speed/` 中的 PyTorch 性能优化技巧（混合精度训练、编译优化等）。

**Q: 训练 loss 降到多少算正常？**
A: 在 "The Verdict" 这种小数据集上，训练 loss 降到 0.5-1.0 就算不错。但小数据集上的训练效果有限——加载 OpenAI 预训练权重后效果会好得多。真实的 LLM 预训练通常在 TB 级文本上进行。

**Q: 困惑度（Perplexity）怎么理解？**
A: `Perplexity = exp(loss)`。直觉上，它表示模型在预测下一个词时，平均在多少个候选词之间犹豫。初始未训练时约为 50,000（≈ 词汇量），训练良好后可降到 2-3。

**Q: 贪婪解码和采样有什么区别？**
A: 贪婪解码（argmax）每步选概率最高的 token，输出确定但单一。采样（multinomial）根据概率分布随机选取，输出多样但可能不够稳定。Temperature 和 top-k 是控制采样随机性的常用手段。

**Q: 为什么加载预训练权重后生成效果好很多？**
A: OpenAI 的 GPT-2 在大规模互联网文本上训练了数周，学习到了丰富的语言知识。我们只在 "The Verdict" 上训练了几分钟，数据量和计算资源都差了几个数量级。加载预训练权重本质上是站在巨人肩膀上。

**Q: 训练数据太少怎么办？**
A: 可以参考 `ch05/03_bonus_pretraining_on_gutenberg/`，在 Project Gutenberg 的全量书籍数据上训练。也可以直接加载 OpenAI 预训练权重，然后在第 6、7 章进行分类微调或指令微调。

<details>
<summary><b>Q15: 语言模型预训练的损失函数是什么？</b></summary>

**A:** 交叉熵损失（Cross-Entropy Loss）。

```python
loss = F.cross_entropy(logits.flatten(0, 1), targets.flatten())
```

对每个位置，模型预测下一个 token 的概率分布，与真实 token 比较计算损失。交叉熵损失等价于负平均对数似然（Negative Average Log-Likelihood）。

</details>

<details>
<summary><b>Q16: 文本生成（贪婪解码）的算法步骤？</b></summary>

**A:**
1. 给定初始提示词 → tokenize → 得到 `idx` `(1, prompt_len)`
2. 循环 `max_new_tokens` 次：
   a. 截取最后 `context_length` 个 token 作为输入：`idx_cond = idx[:, -context_size:]`
   b. `model(idx_cond)` → logits
   c. 取最后一个位置的 logits → `idx_next = argmax(logits[:, -1, :])`
   d. 拼接到 idx 末尾：`idx = torch.cat((idx, idx_next), dim=1)`
3. 解码 idx → 文本

</details>

<details>
<summary><b>Q17: 训练集和验证集的损失趋势说明了什么？</b></summary>

**A:**
- 两者同时下降 → 模型在学习（正常）
- 训练 loss 下降，验证 loss 上升 → 过拟合
- 训练 loss 不降 → 学习率太低或模型有问题
- 验证 loss 远高于训练 loss → 训练数据太少或不具代表性

</details>

<details>
<summary><b>Q18: Temperature 和 Top-k 采样如何影响文本生成？</b></summary>

**A:**
- **Temperature > 1**：概率分布更均匀，生成更随机、更多样
- **Temperature < 1**：概率分布更尖锐，生成更确定、更保守
- **Temperature = 0**：退化为贪婪解码（argmax）
- **Top-k = k**：只从概率最高的 k 个 token 中采样，过滤掉低质量候选

</details>

<details>
<summary><b>Q19: 为什么加载 OpenAI 预训练权重时需要手动映射？</b></summary>

**A:** OpenAI 的 GPT-2 权重使用 TensorFlow 格式存储，且命名约定（如 `c_attn`、`c_proj`、`ln_1`）与我们的 `GPTModel` 不同（如 `W_query`、`out_proj`、`norm1`）。此外，OpenAI 将 QKV 权重合并存储在 `c_attn` 中，需要用 `np.split` 拆分为三个独立的权重矩阵。

</details>

<details>
<summary><b>Q20: `model.eval()` 和 `model.train()` 各自的作用？</b></summary>

**A:**
- `model.train()`：启用 Dropout 和 BatchNorm 的训练行为（随机丢弃、使用 batch 统计量）
- `model.eval()`：禁用 Dropout（不随机丢弃）和 BatchNorm 使用全局统计量。生成文本和评估损失时必须调用，确保结果确定

</details>
