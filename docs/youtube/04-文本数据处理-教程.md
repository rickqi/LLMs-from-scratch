## 一、整体数据流水线

```
原始文本 ("Hello, world.")
    |
    v
[分词 Tokenization]  -->  ['Hello', ',', ' world', '.']
    |
    v
[Token ID 映射]      -->  [15496, 11, 995, 13]

    v

    |
    v
[输入 LLM]
```

## 三、分词 (Tokenization) -- 用正则表达式热身

### 3.1 为什么要分词？

LLM 不能直接处理字符串，必须先将文本拆分为独立的 **Token**（词、子词或标点），再映射为数字。

### 3.2 简单正则分词

```python
import re

text = "Hello, world. This is a test."

# 基础版：按空白字符分割
result = re.split(r'(\s)', text)
print(result)
# ['Hello,', '', ' ', 'world.', '', ' ', 'This', ...]

# 进阶版：将标点也拆为独立 Token
result = re.split(r'([,.:;?_!"()\']|--|\s)', text)
result = [item.strip() for item in result if item.strip()]
print(result)
# ['Hello', ',', 'world', '.', 'This', 'is', 'a', 'test', '.']
```

### 3.3 处理特殊字符

更复杂的文本（包含 `--`、省略号等）需要更完善的正则：

```python
preprocessed = re.split(r'([,.:;?_!"()\']|--|\s)', raw_text)
preprocessed = [item.strip() for item in preprocessed if item.strip()]
print(len(preprocessed))  # 约 4,690 个 Token
```

> 此方法仅为教学热身。实际 LLM 训练使用 BPE 算法（见第五节）。

## 四、构建词汇表与 Token ID

### 4.1 核心概念

```
Token                Token ID (整数)
"the"          -->         7
"Brown"        -->         0
"dog"          -->         1
"fox"          -->         2
...
```

词汇表 = 数据集中所有唯一 Token 到唯一整数的**有序映射**。

### 4.2 构建词汇表

```python
# 去重 + 排序
all_words = sorted(set(preprocessed))
vocab_size = len(all_words)
print(vocab_size)  # 约 1,130

# 构建 Token --> ID 映射
vocab = {token: integer for integer, token in enumerate(all_words)}
```

### 4.3 SimpleTokenizer 类

```python
class SimpleTokenizerV1:
    def __init__(self, vocab):
        self.str_to_int = vocab                          # Token --> ID
        self.int_to_str = {i: s for s, i in vocab.items()}  # ID --> Token

    def encode(self, text):
        preprocessed = re.split(r'([,.:;?_!"()\']|--|\s)', text)
        preprocessed = [item.strip() for item in preprocessed if item.strip()]
        ids = [self.str_to_int[s] for s in preprocessed]
        return ids

    def decode(self, ids):
        text = " ".join([self.int_to_str[i] for i in ids])
        text = re.sub(r'\s+([,.?!"()\'])', r'\1', text)  # 修复标点前空格
        return text
```

使用示例：

```python
tokenizer = SimpleTokenizerV1(vocab)

text = "I HAD always thought Jack Gisburn rather a cheap genius."
ids = tokenizer.encode(text)
print(ids)  # [某个, 一组, 整数, ...]

decoded = tokenizer.decode(ids)
print(decoded)  # "I HAD always thought Jack Gisburn rather a cheap genius."
```

### 4.4 未知词问题

如果输入包含词汇表中没有的词（如 `"hello"`），`encode()` 会抛出 `KeyError`。这是因为 The Verdict 的词汇表太小，不包含所有常见英文单词。

## 五、添加特殊 Token

### 5.1 两种特殊 Token

| Token | 用途 |
|-------|------|
| `<\|unk\|>` | 替代所有不在词汇表中的未知词 |
| `<\|endoftext\|>` | 标记文档边界，分隔不同文档 |

### 5.2 扩展词汇表

```python
all_tokens = sorted(list(set(preprocessed)))
all_tokens.extend(["<|endoftext|>", "<|unk|>"])

vocab = {token: integer for integer, token in enumerate(all_tokens)}
print(len(vocab))  # 1,132（新增 2 个特殊 Token）
```

### 5.3 SimpleTokenizerV2 -- 处理未知词

```python
class SimpleTokenizerV2:
    def __init__(self, vocab):
        self.str_to_int = vocab
        self.int_to_str = {i: s for s, i in vocab.items()}

    def encode(self, text):
        preprocessed = re.split(r'([,.:;?_!"()\']|--|\s)', text)
        preprocessed = [item.strip() for item in preprocessed if item.strip()]
        ids = [self.str_to_int.get(s, self.str_to_int["<|unk|>"])
               for s in preprocessed]
        return ids

    def decode(self, ids):
        text = " ".join([self.int_to_str[i] for i in ids])
        text = re.sub(r'\s+([,.?!"()\'])', r'\1', text)
        return text
```

关键改动：`self.str_to_int.get(s, self.str_to_int["<|unk|>"])` -- 未知词统一映射为 `<|unk|>`。

### 5.4 `<|unk|>` 的局限

```
输入: "Hello, do you like tea? Akwirw ier is a test."
编码: [<|unk|>, ,, do, you, like, tea, ?, <|unk|>, <|unk|>, is, a, test, .]
```

所有未知词变成相同的 `<|unk|>` -- LLM 无法区分 "Hello" 和 "Akwirw ier"。这正是需要 BPE 算法的原因。

## 六、字节对编码 (Byte Pair Encoding, BPE)

### 6.1 BPE 核心思想

```
传统方法:  未知词 --> <|unk|>     (丢失信息)

BPE 方法:  未知词 --> 子词拆分     (保留信息)
           "Akwirw ier" --> ["Ak", "wir", "w", " ier"]
           完全未知的 --> 按字符拆分  ["A", "k", "w", ...]
```

BPE **永远不会失败**：最坏情况按单字符拆分，效率低但不会丢失信息。

### 6.2 谁在使用 BPE？

| 模型 | 分词方法 | 词汇表大小 |
|------|----------|-----------|
| GPT-1 | BPE | -- |
| GPT-2 | BPE | 50,257 |
| GPT-3 | BPE | 50,257 |
| GPT-4 | BPE | ~100,000+ |
| LLaMA 3 | BPE | 128,000 |

### 6.3 使用 tiktoken 库

tiktoken 是 OpenAI 开源的 BPE 分词器，核心代码用 Rust 编写，速度快：

```python
import tiktoken

tokenizer = tiktoken.get_encoding("gpt2")

# 编码
text = "Hello, world. This is a test."
ids = tokenizer.encode(text, allowed_special={"<|endoftext|>"})
print(ids)  # [15496, 11, 995, 13, 770, ...]

# 解码
decoded = tokenizer.decode(ids)
print(decoded)  # "Hello, world. This is a test."
```

### 6.4 `<|endoftext|>` 特殊处理

GPT-2 的 `<|endoftext|>` Token（ID = 50,256）默认不被 tiktoken 接受，需显式启用：

```python
# 不加 allowed_special 会报错
text = "<|endoftext|> Hello world"
ids = tokenizer.encode(text, allowed_special={"<|endoftext|>"})
print(ids)  # [..., 50256, ...]
```

### 6.5 BPE 处理未知词演示

```
已知子词: "sum", "unknown"
输入: "someunknownword"
输出: ["some", "unknown", "word"]     <-- 拆分为已知子词

完全未知: "xyz123"
输出: ["x", "y", "z", "1", "2", "3"]  <-- 回退到单字符
```

> BPE 的完整训练实现见补充材料 `ch02/05_bpe-from-scratch/`，属于高级内容，不影响后续学习。

## 七、滑动窗口数据采样

### 7.1 为什么需要滑动窗口？

LLM 不能一次处理全部 Token。我们需要将长文本切分为固定长度的 **输入-目标对**：

```
输入 (X)        -->  目标 (Y)  (下一个 Token)
--------------------------------------------------
[L L M s]       -->  [L M s learn]
[L M s learn]   -->  [M s learn to]
[M s learn to]  -->  [s learn to predict]
...
```

核心思想：**目标 = 输入右移一位**。LLM 学习的是"给定前面的 Token，预测下一个 Token"。

### 7.2 代码实现

```python
import tiktoken

tokenizer = tiktoken.get_encoding("gpt2")
with open("the-verdict.txt", "r", encoding="utf-8") as f:
    raw_text = f.read()

enc_text = tokenizer.encode(raw_text)
print(len(enc_text))  # 5,145

# 截取前 50 个 Token 用于演示
enc_sample = enc_text[50:]

context_size = 4  # 教学用小窗口; GPT-2 实际使用 1,024

# 构造输入-目标对
for i in range(1, context_size + 1):
    context = enc_sample[:i]
    desired = enc_sample[i]
    print(tokenizer.decode(context), "-->", tokenizer.decode([desired]))
```

输出示例：
```
and -->  established
and established -->  himself
and established himself -->  in
and established himself in -->  a
```

### 7.3 输入与目标的完整构造

```python
import torch

# 输入: Token[0:4]
x = torch.tensor(enc_sample[:context_size])
# 目标: Token[1:5]  (右移一位)
y = torch.tensor(enc_sample[1:context_size+1])

print("Input: ", x)      # tensor([40, 367, 2885, 1464])
print("Target:", y)      # tensor([367, 2885, 1464, 1807])
```

## 八、PyTorch Dataset 与 DataLoader

### 8.1 为什么用 PyTorch DataLoader？

DataLoader 提供：
- 自动 **批处理** (batching)
- 数据 **打乱** (shuffling)
- 多进程并行加载
- 统一的迭代接口

### 8.2 GPTDataset 类

```python
from torch.utils.data import Dataset, DataLoader

class GPTDatasetV1(Dataset):
    def __init__(self, txt, tokenizer, max_length, stride):
        self.input_ids = []
        self.target_ids = []

        token_ids = tokenizer.encode(txt, allowed_special={"<|endoftext|>"})

        # 滑动窗口切分
        for i in range(0, len(token_ids) - max_length, stride):
            input_chunk = token_ids[i:i + max_length]
            target_chunk = token_ids[i + 1: i + max_length + 1]
            self.input_ids.append(torch.tensor(input_chunk))
            self.target_ids.append(torch.tensor(target_chunk))

    def __len__(self):
        return len(self.input_ids)

    def __getitem__(self, idx):
        return self.input_ids[idx], self.target_ids[idx]
```

### 8.3 创建 DataLoader 的辅助函数

```python
def create_dataloader_v1(txt, batch_size=4, max_length=256,
                         stride=128, shuffle=True, drop_last=True,
                         num_workers=0):
    tokenizer = tiktoken.get_encoding("gpt2")
    dataset = GPTDatasetV1(txt, tokenizer, max_length, stride)
    dataloader = DataLoader(
        dataset,
        batch_size=batch_size,
        shuffle=shuffle,
        drop_last=drop_last,
        num_workers=num_workers
    )
    return dataloader
```

### 8.4 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `batch_size` | 4 | 每个 batch 的样本数 |
| `max_length` | 256 | 上下文窗口大小（即 Token 序列长度） |
| `stride` | 128 | 滑动窗口步长（重叠 = max_length - stride） |
| `shuffle` | True | 打乱数据顺序 |
| `drop_last` | True | 丢弃最后一个不完整的 batch |
| `num_workers` | 0 | 数据加载进程数（notebook 中设为 0） |

### 8.5 使用示例

```python
with open("the-verdict.txt", "r", encoding="utf-8") as f:
    raw_text = f.read()

dataloader = create_dataloader_v1(
    raw_text, batch_size=8, max_length=4, stride=4, shuffle=False
)

data_iter = iter(dataloader)
inputs, targets = next(data_iter)
print("Inputs shape: ", inputs.shape)   # torch.Size([8, 4])
print("Targets shape:", targets.shape)  # torch.Size([8, 4])
```

### 8.6 drop_last 的重要性

```
假设数据集有 9 个样本，batch_size = 4:
  Batch 1: [样本 1, 样本 2, 样本 3, 样本 4]  -- 正常
  Batch 2: [样本 5, 样本 6, 样本 7, 样本 8]  -- 正常
  Batch 3: [样本 9]                          -- 只有 1 个!

最后一个不完整的 batch 会导致训练 loss 震荡 (loss spikes)。
设置 drop_last=True 可避免此问题。
```

## 常见问题

**Q1: 为什么不直接按空格分词？**
按空格分词无法处理标点、缩写（如 "don't"）和未知词。BPE 通过子词拆分解决了这些问题。

**Q2: GPT-2 的词汇表有多大？**
50,257 个 Token。其中 ID 0-50,255 是常规 Token，50,256 是 `<|endoftext|>`。

**Q3: stride 和 max_length 的关系是什么？**
- `stride = max_length`：无重叠，每个窗口完全独立
- `stride < max_length`：有重叠，增加训练样本数量，但占用更多内存

**Q4: 为什么用 tiktoken 而不是自己实现 BPE？**
tiktoken 的核心用 Rust 编写，比纯 Python 实现快很多。分词器不属于 LLM 本身，属于预处理工具，使用现成库是工程最佳实践。

**Q5: 如果训练数据有几万亿 Token 怎么办？**
本教程的方法（预存所有 input-target 对）不适合超大数据集。工业界使用流式加载、内存映射等技术。但对教育用途（几百 MB 到几 GB），当前方法完全够用。
