## 第一步：检查现有的 Python

打开终端，检查是否已安装 Python：

```bash
python --version
# 或
python3 --version
```

**三种可能结果：**

| 结果 | 含义 | 做法 |
|------|------|------|
| `command not found` | 没装 Python | 去下一步安装 |
| 显示旧版本（如 3.9） | 系统自带 Python | **不要用它！** 另装一个独立版本 |
| 显示 3.10+ | 已有合适版本 | 可以跳过安装步骤 |

> ⚠️ **绝对不要修改系统自带的 Python。** 操作系统用它来运行自己的工具，乱改可能会搞坏系统。

## 第三步：安装 UV 包管理器

### 为什么用 UV 而不是 pip？

| 工具 | 速度 | 特点 |
|------|------|------|
| **UV** | ⚡ 极快 | Rust 编写，比 pip 快 10-100 倍，语法兼容 pip |
| pip | 🐢 一般 | Python 官方内置 |
| conda | 🐢 较慢 | 科学计算全家桶 |

UV 是 Python 社区当前最热门的包管理器，大量开发者已迁移。本教程使用 `uv pip` 语法（和 pip 几乎一样）。

## 第五步：创建虚拟环境

**虚拟环境是什么？** 一个独立的文件夹，里面有自己的 Python + 所有包。搞坏了直接删掉重建，不影响系统和其它项目。

```bash
# 在项目根目录下执行
uv venv --python 3.11
```

这会在当前目录创建一个 `.venv` 文件夹：

```
LLMs-from-scratch/
├── .venv/               ← 虚拟环境（可随时删除重建）
│   ├── bin/python        ← 独立的 Python 3.11
│   └── lib/.../site-packages/  ← 所有安装的包
├── ch02/
├── ch03/
└── ...
```

> 💡 `.venv` 以 `.` 开头，在文件管理器中默认隐藏。macOS Finder 需要启用"显示隐藏文件"。

## 第七步-B：AMD GPU + WSL2 安装 ROCm 版 PyTorch

> 🎯 适用场景：AMD 显卡 + Windows 11 WSL2（Ubuntu 22.04/24.04）

### B.1 前置条件检查

| 要求 | 最低版本 | 检查命令 |
|------|---------|---------|
| Windows | Windows 11 **24H2** (Build 26100+) | `winver`（Windows 运行） |
| AMD 驱动 | Adrenalin **26.2.2+** | 打开 AMD Adrenalin 软件查看 |
| **Windows SDK** ⚠️ | **10.0.26100+** | Windows 端下载安装（见下方） |
| **WSL2 内核** ⚠️ | **6.6+**（推荐 6.18+） | `uname -r` |
| Ubuntu | 22.04 或 24.04 | `lsb_release -a` |
| Python | 3.10（22.04）/ 3.12（24.04） | `python3 --version` |

> ⚠️ **Windows SDK 是强制前置条件！** 缺少则无法构建 librocdxg，GPU 完全不可用。
> 下载地址：https://developer.microsoft.com/en-us/windows/downloads/windows-sdk/
> 安装时只需勾选 **"Windows SDK for Desktop C++"**。

> ⚠️ **WSL2 内核版本 ≥ 6.6 是 AMD GPU 透传的前提！** 旧内核（如 5.15）下 `/dev/kfd` 不会出现，ROCm 完全无法识别 GPU。
> 更新方法（**Windows PowerShell 管理员**）：
> ```powershell
> wsl --update
> wsl --shutdown    # 必须重启 WSL2 才能生效！
> ```
> 然后重新打开 WSL2 终端，用 `uname -r` 确认内核版本已更新。

### B.2 支持的 AMD GPU 列表

> ⚠️ **仅 RDNA3 及以上架构支持 WSL2 GPU 加速。** 老显卡（RX 6000 及以前）不支持。

| ✅ 支持的 GPU | ❌ 不支持的 GPU |
|---|---|
| RX 7900 XTX / 7900 XT / 7900 GRE | RX 6000 系列及更早 |
| RX 7800 XT / 7700 XT | RX 5700 XT 及更早 |
| RX 9070 XT / 9070 / 9060 XT / 9060 | 老 APU（Vega 架构） |
| Radeon 890M / 880M / 860M (RDNA 3.5) | |
| Ryzen AI 9 HX 370 / AI 9 365 | |

### B.3 安装步骤

**1. Windows 端：更新驱动（如需要）**

```
从 AMD 官网下载 Adrenalin 26.2.2+ 驱动 → 安装 → 重启
```

**1b. Windows 端：安装 Windows SDK**（⚠️ 必须！）

1. 浏览器打开 https://developer.microsoft.com/en-us/windows/downloads/windows-sdk/
2. 下载最新 SDK 安装器 → 运行
3. 安装时只需勾选 **"Windows SDK for Desktop C++"**（其余可选）
4. 完成后重启 WSL2：PowerShell 中执行 `wsl --shutdown`

**2. WSL2 端：升级/安装 ROCm 7.2.4**

```bash
# 如果已有旧版 ROCm，先清理
sudo apt remove --purge rocm-* amdgpu-* 2>/dev/null
sudo apt autoremove -y
sudo rm -rf /opt/rocm-*

# 安装 amdgpu-install（latest = 7.2.4）
wget https://repo.radeon.com/amdgpu-install/latest/ubuntu/jammy/amdgpu-install_7.2.4.70204-1_all.deb
sudo apt install ./amdgpu-install_7.2.4.70204-1_all.deb

# 安装最小 ROCm 运行时（推荐，避免下载 6GB 完整包）
sudo apt-get update
sudo apt-get install -y --no-install-recommends \
  rocm-core libhsa-runtime64-1 rocminfo hip-runtime-amd \
  rocm-device-libs rocblas comgr hsa-rocr-dev
```

> 💡 上述命令安装 ~2GB 的最小运行时。如需完整 ROCm（~27GB），改用 `sudo amdgpu-install --usecase=rocm --no-dkms --no-32 -y`。

**2b. 构建 librocdxg**（ROCm 7.2+ 在 WSL2 的必需组件）

```bash
# 确认 Windows SDK 已安装且版本 ≥ 10.0.26100
ls "/mnt/c/Program Files (x86)/Windows Kits/10/Include/"
# 应该看到 10.0.26100.0 或更新版本

# 克隆并构建
git clone https://github.com/ROCm/librocdxg.git /tmp/librocdxg
cd /tmp/librocdxg
export win_sdk="/mnt/c/Program Files (x86)/Windows Kits/10/Include/10.0.26100.0/"
mkdir -p build && cd build
cmake .. -DWIN_SDK="${win_sdk}/shared"
make -j$(nproc)
sudo make install
cd ~

# 验证
LD_LIBRARY_PATH=/opt/rocm/lib:$LD_LIBRARY_PATH rocminfo
# 应显示你的 AMD GPU 信息
```

> ⚠️ 若编译报错 `D3DKMT_QUERYSTATISTICS_SEGMENT_TYPE_SYSMEM` 等符号未定义，说明 Windows SDK 版本过旧。必须 ≥ 10.0.26100。

**3. 设置环境变量**

```bash
# 追加到 ~/.bashrc（RDNA 3.5 用 11.5.0，RDNA 3 用 11.0.0）
echo 'export HSA_OVERRIDE_GFX_VERSION=11.5.0' >> ~/.bashrc
echo 'export HIP_DEVICE_ORDER=PCI_BUS_ID' >> ~/.bashrc
echo 'export HSA_ENABLE_DXG_DETECTION=1' >> ~/.bashrc
echo 'export LD_LIBRARY_PATH=/opt/rocm/lib:$LD_LIBRARY_PATH' >> ~/.bashrc
echo 'export PATH=/opt/rocm/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

> 💡 **`HSA_OVERRIDE_GFX_VERSION` 对照表：**
>
> | GPU 架构 | 值 | 代表型号 |
> |---------|-----|---------|
> | RDNA 3 | `11.0.0` | RX 7900 XTX/XT, 7900 GRE |
> | RDNA 3.5 | `11.5.0` | Radeon 890M/880M, Ryzen AI 9 HX 370 |
> | RDNA 4 | `12.0.0` | RX 9070 XT/9070 |

**4. 验证 ROCm 可识别 GPU**

> ⚠️ **安装 PyTorch 前必须先确认 ROCm 能看到 GPU！** 如果 `rocminfo` 不显示 GPU 信息，PyTorch 也无法使用 GPU。

```bash
# 验证 /dev/kfd 存在（AMD GPU 透传的关键标志）
ls /dev/kfd
# 应输出: /dev/kfd

# 验证 ROCm 可识别 GPU
rocminfo | grep -E "Name:|Marketing|gfx"
# 应显示你的 AMD GPU 型号和 gfx 架构
```

> ❌ 如果 `/dev/kfd` 不存在，说明 GPU 没有从 Windows 透传到 WSL2。按顺序排查：
> 1. Windows 版本是否为 24H2 (Build 26100+) → `winver` 检查
> 2. AMD 驱动是否 ≥ 26.2.2 → 打开 Adrenalin 查看
> 3. WSL2 内核是否 ≥ 6.6 → `uname -r` 检查
> 4. 是否执行了 `wsl --shutdown` 后重开 WSL2
> 5. 你的 GPU 是否是 RDNA3+ 架构（见上方兼容列表）

**5. 安装 ROCm 版 PyTorch**

```bash
# 激活虚拟环境后执行
source .venv/bin/activate

# 先下载匹配的 triton wheel（PyTorch 的必需依赖）
# （以 Ubuntu 22.04 / Python 3.10 为例）
cd /tmp
wget https://repo.radeon.com/rocm/manylinux/rocm-rel-7.2/triton-3.5.1%2Brocm7.2.0.gita272dfa8-cp310-cp310-linux_x86_64.whl

# 再下载 ROCm 版 PyTorch wheel
wget https://repo.radeon.com/rocm/manylinux/rocm-rel-7.2/torch-2.9.1%2Brocm7.2.0.lw.git7e1940d4-cp310-cp310-linux_x86_64.whl

# 先装 triton，再装 torch（顺序重要！）
pip install triton-3.5.1+rocm7.2.0.gita272dfa8-cp310-cp310-linux_x86_64.whl
pip install torch-2.9.1+rocm7.2.0.lw.git7e1940d4-cp310-cp310-linux_x86_64.whl

rm triton-*.whl torch-*.whl
cd -
```

> ⚠️ **不要用 pytorch.org 的 ROCm wheel**——AMD 官方建议使用 `repo.radeon.com` 的专用版本。
>
> **必须先安装 ROCm 版 triton**，否则 PyTorch wheel 安装时会因找不到匹配的 triton 版本而失败（报错 `No matching distribution found for triton==3.5.1+rocm7.2.0...`）。
>
> **其他 Python 版本**：将 wheel 文件名中的 `cp310` 替换为 `cp311`（Python 3.11）或 `cp312`（Python 3.12）。完整 wheel 列表见：https://repo.radeon.com/rocm/manylinux/rocm-rel-7.2/
>
> ⚠️ **安装完 ROCm 版 PyTorch 后，永远不要运行 `uv pip install -r requirements.txt`**——它会用 CPU/CUDA 版覆盖你的 ROCm 版 PyTorch。需要补装其他包时用 `uv pip install <包名>` 单独安装。

**6. 验证 GPU 可用**

```bash
python -c "import torch; print('ROCm available:', torch.cuda.is_available())"
python -c "import torch; print('GPU count:', torch.cuda.device_count())"
python -c "import torch; print('GPU name:', torch.cuda.get_device_name(0) if torch.cuda.is_available() else 'N/A')"
```

期望输出：
```
ROCm available: True
GPU count: 1
GPU name: AMD Radeon(TM) 890M Graphics
```

> 💡 ROCm 版 PyTorch 中 `torch.cuda` 自动映射到 AMD GPU——**代码完全不需要修改**。

**7. 安装其余依赖**

```bash
# 单独安装（跳过 torch，因为已手动安装）
uv pip install jupyterlab tiktoken matplotlib tensorflow tqdm pandas
```

> ⚠️ **不要在 AMD 环境下运行 `uv pip install -r requirements.txt`**——它会覆盖你刚装的 ROCm 版 PyTorch。

### B.4 已知问题

| 问题 | 解决方案 |
|------|---------|
| `librocdxg.so: cannot open` | Windows SDK 未安装或版本过旧。安装 SDK ≥ 10.0.26100 后重新构建 librocdxg |
| `hsaKmtOpenKFD undefined` | 同上 — librocdxg 未正确安装。WSL2 必须走 DXG 桥接 |
| `rocminfo` 报 `hsa_init Failed` | WSL2 内核过旧或 GPU 未透传。检查：(1) `uname -r` 应 ≥ 6.6；(2) `/dev/kfd` 应存在；(3) PowerShell 执行 `wsl --update && wsl --shutdown` 后重开 |
| `rocminfo` segfault / 找不到 GPU | 检查 `HSA_OVERRIDE_GFX_VERSION` 是否匹配、`HSA_ENABLE_DXG_DETECTION=1` 是否设置 |
| `torch.cuda.is_available()` 返回 `False` | (1) 确认 `rocminfo` 能看到 GPU；(2) 确认 librocdxg 已构建安装；(3) 确认 PyTorch 是 ROCm 版（`pip show torch \| grep rocm`）；(4) 确认 triton 也是 ROCm 版 |
| PyTorch 安装报错 `No matching distribution found for triton` | 需要先从 `repo.radeon.com` 下载并安装匹配的 ROCm 版 triton wheel，再安装 PyTorch |
| `pip install torch wheel` 文件名无效 | wget 下载时用 `-O` 重命名会丢失 wheel 元数据。保持原始文件名即可 |
| numpy 版本冲突 | `pip install numpy==1.26.4` |
| 驱动更新后 GPU 不可用 | PowerShell 执行 `wsl --shutdown` 后重开 |

## 完整操作流程（TL;DR）

### NVIDIA GPU / 无 GPU 用户

```bash
# 1. 克隆代码
git clone --depth 1 https://github.com/rasbt/LLMs-from-scratch.git
cd LLMs-from-scratch

# 2. 安装 UV
pip install uv

# 3. 创建虚拟环境
uv venv --python 3.11

# 4. 激活虚拟环境
source .venv/bin/activate   # Windows: .venv\Scripts\activate

# 5. 安装依赖
uv pip install -r requirements.txt

# 6. 验证安装
python -c "import torch; print(f'PyTorch {torch.__version__}')"
python -c "import tiktoken; print(f'tiktoken {tiktoken.__version__}')"

# 7. 启动 Jupyter Lab
jupyter lab
```

### AMD GPU + WSL2 用户

```bash
# 0. ⚠️ Windows 端前置：安装 AMD 驱动 ≥ 26.2.2 + Windows SDK ≥ 10.0.26100+
#    详见教程 → 第七步-B → B.1 前置条件检查

# 1. 克隆代码、安装 UV、创建虚拟环境（同 NVIDIA 流程 1-4）

# 2. 更新 WSL2 内核（Windows PowerShell 管理员执行）
#    wsl --update
#    wsl --shutdown    # 必须重启 WSL2！
#    然后重新打开 WSL2，用 uname -r 确认内核 ≥ 6.6

# 3. 安装 ROCm 7.2.4 + 构建 librocdxg（详见教程第七步-B.3，约 15 分钟）

# 4. 设置 AMD 环境变量
echo 'export HSA_OVERRIDE_GFX_VERSION=11.5.0' >> ~/.bashrc
echo 'export HIP_DEVICE_ORDER=PCI_BUS_ID' >> ~/.bashrc
echo 'export HSA_ENABLE_DXG_DETECTION=1' >> ~/.bashrc
echo 'export PATH=/opt/rocm/bin:$PATH' >> ~/.bashrc
echo 'export LD_LIBRARY_PATH=/opt/rocm/lib:$LD_LIBRARY_PATH' >> ~/.bashrc
source ~/.bashrc

# 5. 验证 GPU 透传（⚠️ 必须通过才能继续！）
ls /dev/kfd          # 应存在
rocminfo             # 应显示 AMD GPU 信息

# 6. 安装 ROCm 版 PyTorch（先 triton，再 torch）
cd /tmp
wget https://repo.radeon.com/rocm/manylinux/rocm-rel-7.2/triton-3.5.1%2Brocm7.2.0.gita272dfa8-cp310-cp310-linux_x86_64.whl
wget https://repo.radeon.com/rocm/manylinux/rocm-rel-7.2/torch-2.9.1%2Brocm7.2.0.lw.git7e1940d4-cp310-cp310-linux_x86_64.whl
pip install triton-3.5.1+rocm7.2.0.gita272dfa8-cp310-cp310-linux_x86_64.whl
pip install torch-2.9.1+rocm7.2.0.lw.git7e1940d4-cp310-cp310-linux_x86_64.whl
rm triton-*.whl torch-*.whl
cd -

# 7. 安装其余依赖（跳过 torch）
uv pip install jupyterlab tiktoken matplotlib tensorflow tqdm pandas

# 8. 验证 GPU
python -c "import torch; print('GPU:', torch.cuda.is_available(), torch.cuda.get_device_name(0) if torch.cuda.is_available() else '')"

# 9. 启动 Jupyter Lab
jupyter lab
```

## 常见问题

**Q: `python` 命令找不到，只有 `python3`？**
A: 激活虚拟环境后，`python` 命令会自动指向虚拟环境中的 Python。

**Q: 虚拟环境激活后需要退出吗？**
A: 用完可以输入 `deactivate` 退出，或直接关闭终端。

**Q: `uv` 安装包报错？**
A: 回退用 `pip install -r requirements.txt`，功能相同只是慢一些。

**Q: 没有 GPU 能跑吗？**
A: 完全可以。第 2~4 章不需要 GPU，第 5 章训练用 Colab 免费 T4 GPU。

**Q: macOS 上 `.venv` 文件夹不见了？**
A: 以 `.` 开头默认隐藏。Finder 中按 `Cmd + Shift + .` 切换显示。

**Q: 我是 AMD 显卡，`torch.cuda.is_available()` 返回 `False`？**
A: 按顺序排查：(1) `rocminfo` 是否能看到 GPU（如果看不到，GPU 没有透传到 WSL2）；(2) `/dev/kfd` 是否存在（不存在则内核过旧或未 `wsl --shutdown`）；(3) `uname -r` 是否 ≥ 6.6；(4) Windows SDK ≥ 10.0.26100 是否已安装；(5) librocdxg 是否已构建安装；(6) `HSA_OVERRIDE_GFX_VERSION` 是否匹配 GPU 架构；(7) PyTorch 和 triton 是否都是 ROCm 版（`pip show torch | grep rocm` 和 `pip show triton | grep rocm` 应有输出）。

**Q: `rocminfo` 报 `hsa_init Failed, possibly no supported GPU devices`？**
A: GPU 没有从 Windows 透传到 WSL2。按顺序排查：(1) `uname -r` 检查内核版本，应 ≥ 6.6（旧内核如 5.15 不支持 AMD GPU 透传）；(2) 在 Windows PowerShell（管理员）中执行 `wsl --update && wsl --shutdown`，然后重新打开 WSL2；(3) 确认 AMD 驱动 ≥ 26.2.2；(4) 确认 Windows 为 24H2 (Build 26100+)；(5) 确认你的 GPU 是 RDNA3+ 架构。

**Q: rocminfo 报错 `librocdxg.so: cannot open`？**
A: librocdxg 未构建。ROCm 7.2+ 在 WSL2 需要此库桥接 Windows 驱动。安装 Windows SDK 10.0.26100+ 后按教程第七步-B.2b 构建。

**Q: AMD GPU 只有 4GB VRAM，够用吗？**
A: 第 2~4 章完全够用。第 5 章 124M GPT 模型训练可以跑，但 batch size 会受限。建议大训练用 Colab。

**Q: WSL2 中 AMD GPU 时灵时不灵？**
A: Windows 更新驱动后 WSL2 的 GPU 直通可能断开。PowerShell 执行 `wsl --shutdown` 后重开即可。

**Q: 如何确认我的 AMD 显卡是否支持 WSL2 GPU 加速？**
A: RDNA3 及以上架构（RX 7000 系列、RX 9000 系列、Radeon 800M 系列）支持。RX 6000 及更老型号不支持。详见 [第七步-B 的兼容列表](#b2-支持的-amd-gpu-列表)。
