## 三、文本生成工具函数

### 3.1 两个便利函数

```python
def text_to_token_ids(text, tokenizer):
    """将文本转为 token IDs（含 batch 维度）"""
    encoded = tokenizer.encode(text)
    encoded_tensor = torch.tensor(encoded).unsqueeze(0)  # 添加 batch 维度
    return encoded_tensor

def token_ids_to_text(token_ids, tokenizer):
    """将 token IDs 转回文本（去除 batch 维度）"""
    flat = token_ids.squeeze(0)  # 移除 batch 维度
    return tokenizer.decode(flat.tolist())
```

### 3.2 生成流程

```
输入文本
    |
    v
text_to_token_ids()  -->  token IDs (tensor)
    |
    v
GPT 模型前向传播     -->  logits [batch, seq_len, vocab_size]
    |
    v
取最后一个位置的 logits --> 概率分布 (softmax)
    |
    v
argmax 或采样        -->  下一个 token ID
    |
    v
token_ids_to_text()  -->  生成文本
```

### 3.3 简单生成函数（来自 Ch.4）

```python
def generate_text_simple(model, idx, max_new_tokens, context_size):
    """贪心解码：每步取概率最高的 token"""
    for _ in range(max_new_tokens):
        idx_cond = idx[:, -context_size:]  # 截取上下文窗口
        with torch.no_grad():
            logits = model(idx_cond)
        logits = logits[:, -1, :]  # 只看最后一个位置
        idx_next = torch.argmax(logits, dim=-1, keepdim=True)  # 贪心选择
        idx = torch.cat((idx, idx_next), dim=1)  # 拼接
    return idx
```

### 3.4 未训练模型的输出

```python
tokenizer = tiktoken.get_encoding("gpt2")
token_ids = generate_text_simple(
    model=model,
    idx=text_to_token_ids("Hello, I am", tokenizer),
    max_new_tokens=10,
    context_size=GPT_CONFIG_124M["context_length"]
)
print(token_ids_to_text(token_ids, tokenizer))
# 输出类似: "Hello, I am armt Netflix..." —— 乱码（因为模型未训练）
```

> 这正是我们需要训练的原因：模型用随机权重初始化，不知道如何生成有意义的文本。

## 四、交叉熵损失函数（Cross-Entropy Loss）

### 4.1 为什么需要损失函数？

人类看输出文本说"这是乱码"，但计算机需要一个**数值化度量**来衡量生成质量，才能通过优化算法来改善它。

### 4.2 输入与目标的对齐

在第 2 章中我们设置过：**目标（targets）比输入（inputs）偏移一个位置**：

```
输入:  [every]  [effort]  [moves]
目标:  [effort] [moves]   [下一个token]

输入:  [I]      [really]  [like]
目标:  [really] [like]    [chocolate]
```

> 两个 batch 行之间彼此无关（可以来自完全不同的文本段落）。关系只存在于同一行内：输入的第 i 个位置对应目标的第 i 个位置。

### 4.3 Logits 与概率

模型输出的 logits 形状为 `[batch_size, seq_len, vocab_size]`：

```
输入 "every effort moves" (3 个 token, vocab_size=50257)
    |
    v
GPT 模型
    |
    v
logits: shape [1, 3, 50257]
    |
    v
softmax
    |
    v
probas: shape [1, 3, 50257]  -- 每行是 50257 个概率值，总和为 1
```

每个位置的概率向量告诉我们：模型认为下一个 token 是词汇表中每个词的可能性。

### 4.4 手动计算交叉熵损失

逐步分解（以教学为目的）：

```python
# 1. 获取 logits
logits = model(inputs)                # shape: [2, 3, 50257]

# 2. 计算 softmax 概率
probas = torch.softmax(logits, dim=-1) # shape: [2, 3, 50257]

# 3. 提取目标位置的概率
#    对每个位置，查找目标 token ID 对应的概率值
target_probas_1 = probas[0, 0, targets[0, 0]]  # 第1个位置的目标概率
target_probas_2 = probas[0, 1, targets[0, 1]]  # 第2个位置的目标概率
target_probas_3 = probas[0, 2, targets[0, 2]]  # 第3个位置的目标概率

# 4. 取对数（log probabilities）
log_probas = torch.log(torch.cat([target_probas_1, target_probas_2, target_probas_3]))

# 5. 计算平均
avg_log_probas = torch.mean(log_probas)   # 例如: -10.79

# 6. 取负得到交叉熵损失
cross_entropy_loss = -avg_log_probas      # 例如: 10.79
```

### 4.5 关键直觉

```
概率接近 1  -->  log 接近 0     -->  损失接近 0  (好)
概率接近 0  -->  log 非常负    -->  损失非常大  (差)
```

**训练目标**：让目标 token 对应的概率尽可能大（接近 1），从而使损失接近 0。

### 4.6 PyTorch 内置函数（推荐）

手动步骤可以用一行代码替代：

```python
import torch.nn.functional as F

loss = F.cross_entropy(logits.flatten(0, 1), targets.flatten())
# 结果相同: 10.79
```

> `F.cross_entropy` 内部自动完成 softmax + log + 选择目标位置 + 平均 + 取负，且经过数值优化（GPU 上的 CUDA kernel 加速）。

### 4.7 为什么要 flatten？

```
logits:   [2, 3, 50257]  --(flatten(0,1))-->  [6, 50257]
targets:  [2, 3]         --(flatten())----->  [6]
```

`F.cross_entropy` 期望 logits 是二维的 `[N, vocab_size]`，所以要把 batch 和 seq_len 维度合并。

### 4.8 交叉熵的同义词

| 名称 | 来源 |
|------|------|
| **交叉熵损失 (Cross-Entropy Loss)** | 信息论 / 计算机科学 |
| **负平均对数似然 (Negative Average Log-Likelihood)** | 统计学 |
| **负平均对数概率 (Negative Average Log Probability)** | 统计学 |

> 它们是同一个东西，只是不同领域独立推导出来的。PyTorch 使用 "cross_entropy" 这个名称。

### 4.9 困惑度 (Perplexity)

```python
perplexity = torch.exp(cross_entropy_loss)
```

| 指标 | 初始值（未训练） | 训练后目标 |
|------|------------------|------------|
| 交叉熵损失 | ~10.98 | ~0.5 以下 |
| 困惑度 | ~58,000 | ~1.6 |

> 困惑度可以理解为"模型在多少个词之间犹豫不决"。初始时困惑度约等于词汇表大小（50,257），训练后期望降到 1-2。

### 5.6 损失计算函数

```python
def calc_loss_batch(input_batch, target_batch, model, device):
    """计算单个 batch 的损失"""
    input_batch = input_batch.to(device)
    target_batch = target_batch.to(device)
    logits = model(input_batch)
    loss = torch.nn.functional.cross_entropy(
        logits.flatten(0, 1), target_batch.flatten()
    )
    return loss


def calc_loss_loader(data_loader, model, device, num_batches=None):
    """计算整个 DataLoader 的平均损失"""
    total_loss = 0.
    if len(data_loader) == 0:
        return float("nan")
    if num_batches is None:
        num_batches = len(data_loader)
    else:
        num_batches = min(num_batches, len(data_loader))
    for i, (input_batch, target_batch) in enumerate(data_loader):
        if i >= num_batches:
            break
        loss = calc_loss_batch(input_batch, target_batch, model, device)
        total_loss += loss.item()
    return total_loss / num_batches


def evaluate_model(model, train_loader, val_loader, device, eval_iter):
    """评估训练集和验证集损失"""
    model.eval()
    with torch.no_grad():
        train_loss = calc_loss_loader(train_loader, model, device, num_batches=eval_iter)
        val_loss = calc_loss_loader(val_loader, model, device, num_batches=eval_iter)
    model.train()
    return train_loss, val_loss
```

> `model.eval()` + `torch.no_grad()` + `model.train()` 的组合确保评估时不影响训练状态。

## 六、训练循环

### 6.1 核心训练循环

```python
def train_model_simple(model, train_loader, val_loader, optimizer, device,
                       num_epochs, eval_freq, eval_iter, start_context, tokenizer):
    train_losses, val_losses, track_tokens_seen = [], [], []
    tokens_seen = 0
    global_step = -1

    for epoch in range(num_epochs):
        model.train()

        for input_batch, target_batch in train_loader:
            optimizer.zero_grad()       # 1. 清零梯度
            loss = calc_loss_batch(     # 2. 前向传播 + 计算损失
                input_batch, target_batch, model, device
            )
            loss.backward()             # 3. 反向传播（计算梯度）
            optimizer.step()            # 4. 更新权重
            tokens_seen += input_batch.numel()
            global_step += 1

            if global_step % eval_freq == 0:
                train_loss, val_loss = evaluate_model(
                    model, train_loader, val_loader, device, eval_iter
                )
                train_losses.append(train_loss)
                val_losses.append(val_loss)
                track_tokens_seen.append(tokens_seen)
                print(f"Ep {epoch+1} (Step {global_step:06d}): "
                      f"Train loss {train_loss:.3f}, Val loss {val_loss:.3f}")

        generate_and_print_sample(model, tokenizer, device, start_context)

    return train_losses, val_losses, track_tokens_seen
```

### 6.2 训练循环流程图

```
for epoch in num_epochs:
    |
    v
model.train()  ────── 启用 Dropout 等训练特性
    |
    v
for batch in train_loader:
    |
    +---> optimizer.zero_grad()  ── 清零上一步的梯度
    |
    +---> loss = forward(batch)  ── 前向传播 + 计算损失
    |
    +---> loss.backward()        ── 反向传播（自动微分计算梯度）
    |
    +---> optimizer.step()       ── 根据梯度更新模型权重
    |
    +---> [定期评估] evaluate_model()  ── 在训练集/验证集上计算损失
    |
    v
[每个 epoch 结束] generate_and_print_sample()  ── 生成示例文本观察效果
```

## 八、解码策略：温度与 Top-K 采样

### 8.1 为什么需要采样？

`generate_text_simple` 使用 **argmax（贪心解码）**，总是选择概率最高的 token。这导致：
- 输出确定性（每次相同）
- 容易"记住"训练数据
- 缺乏多样性

> 所有 LLM 服务（ChatGPT 等）底层都使用温度和 Top-K/Top-P 采样来增加多样性。

### 8.2 `torch.multinomial` 概率采样

```python
# 不用 argmax，而是根据概率分布随机采样
probas = torch.softmax(logits, dim=-1)
idx_next = torch.multinomial(probas, num_samples=1)
```

示例：假设概率分布为 `forward: 58%, closer: 7%, on to board: 34%, inches: 1%`

```python
# 运行 1000 次 multinomial 采样的结果：
# forward: 582/1000, on to board: 343/1000, closer: 73/1000, inches: 2/1000
```

### 8.3 温度缩放 (Temperature Scaling)

```python
def softmax_with_temperature(logits, temperature):
    """温度缩放：除以 temperature 后再 softmax"""
    scaled_logits = logits / temperature
    return torch.softmax(scaled_logits, dim=-1)
```

```
温度值    |  效果             |  分布形态
---------|------------------|----------
T < 1    |  更确定（更尖锐）  |  高概率 token 被进一步放大
         |  如 T=0.1        |  forward: 58% -> 95%
T = 1    |  不变             |  保持原始分布
T > 1    |  更随机（更平坦）  |  概率趋于均匀
         |  如 T=5          |  forward: 58% -> 20%, 其他概率均上升
```

> 温度不改变 softmax 的性质（概率总和仍为 1），只改变分布的"锐度"。

### 8.4 Top-K 采样

```python
if top_k is not None:
    # 只保留概率最高的 K 个 token
    top_logits, _ = torch.topk(logits, top_k)
    min_val = top_logits[:, -1]  # 第 K 大的值
    logits = torch.where(
        logits < min_val,
        torch.tensor(float("-inf")).to(logits.device),  # 其余设为 -inf
        logits
    )
```

Top-K 的工作原理：

```
原始 logits:   [1.2, 0.5, -0.3, 3.1, 0.8, -1.5, 0.1]
                                   ^
Top-K=3 保留:  [-inf, -inf, -inf, 3.1, 0.8, -inf, -inf]
                                   ↑最大  ↑第二  ↑第三
softmax 后:    [0, 0, 0, 0.71, 0.16, 0, 0.13]
```

### 8.5 完整的生成函数

```python
def generate(model, idx, max_new_tokens, context_size,
             temperature=0.0, top_k=None, eos_id=None):
    for _ in range(max_new_tokens):
        idx_cond = idx[:, -context_size:]
        with torch.no_grad():
            logits = model(idx_cond)
        logits = logits[:, -1, :]

        # Top-K 过滤
        if top_k is not None:
            top_logits, _ = torch.topk(logits, top_k)
            min_val = top_logits[:, -1]
            logits = torch.where(
                logits < min_val,
                torch.tensor(float("-inf")).to(logits.device),
                logits
            )

        # 温度缩放 + 采样
        if temperature > 0.0:
            logits = logits / temperature
            logits = logits - logits.max(dim=-1, keepdim=True).values  # 数值稳定性
            probs = torch.softmax(logits, dim=-1)
            idx_next = torch.multinomial(probs, num_samples=1)
        else:
            idx_next = torch.argmax(logits, dim=-1, keepdim=True)  # 贪心

        if eos_id is not None and idx_next == eos_id:
            break

        idx = torch.cat((idx, idx_next), dim=1)

    return idx
```

### 8.6 使用示例

```python
token_ids = generate(
    model=gpt,
    idx=text_to_token_ids("Every effort moves you", tokenizer).to(device),
    max_new_tokens=25,
    context_size=GPT_CONFIG_124M["context_length"],
    top_k=50,
    temperature=1.0
)
```

| 参数组合 | 效果 |
|----------|------|

## 关键概念速查表

| 概念 | 一句话解释 |
|------|-----------|
| 交叉熵损失 | 衡量模型预测与真实目标的差距，训练目标是使其接近零 |
| 困惑度 | `exp(交叉熵)`，表示模型在多少个词之间犹豫 |
| Logits | 模型输出的原始分数向量，形状为 `[batch, seq, vocab_size]` |
| Softmax | 将 logits 转为概率分布（总和为 1） |
| Argmax | 选择概率最高的 token（贪心解码） |
| 温度 | 控制采样随机性，T<1 更确定，T>1 更随机 |
| Top-K | 只从概率最高的 K 个 token 中采样 |
| Epoch | 遍历一次完整训练数据集 |
| 学习率 | 控制每次权重更新的步长大小 |
| AdamW | 带权重衰减的 Adam 优化器 |
| `model.eval()` | 切换到推理模式（禁用 Dropout） |
| `torch.no_grad()` | 禁用梯度计算，节省内存和计算 |
| DataLoader | 批量加载数据的工具，支持打乱和并行 |
| 过拟合 | 训练损失下降但验证损失不再改善 |
| Dropout | 训练时随机丢弃神经元以防止过拟合 |
| `state_dict` | PyTorch 模型的参数字典，用于保存/加载 |

## 常见问题

**Q1: 为什么初始损失约为 10.79？**

这是 `-log(1/vocab_size) = -log(1/50257) ≈ 10.82`。未训练模型对所有词汇表条目赋予大致均匀的概率，所以每个 token 的预测概率约为 1/50257。

**Q2: 训练损失降到很低但验证损失不降了，怎么办？**

这是过拟合的信号。解决方法：增大训练数据、减小模型规模、增加正则化（如增大 dropout）、或使用早停（early stopping）。本例中数据集极小（仅 ~5000 token），所以过拟合是预期的。

**Q3: `stride=context_length` 和 `stride=1` 有什么区别？**

`stride=context_length` 产生不重叠的训练样本；`stride=1` 产生最大重叠（每个位置都是一个训练样本），训练数据更多但计算量也更大。实际预训练通常用较大的 stride。

**Q4: 温度和 Top-K 应该怎么设置？**

没有固定答案。一般：温度 0.7-1.2 较常见，Top-K 40-100 较常见。温度越低输出越确定但越无聊，越高越多样但可能不连贯。Top-K 越小输出越安全但越单调。

**Q5: 为什么要用 TensorFlow 来加载 OpenAI 权重？**

OpenAI 最初用 TensorFlow 训练 GPT-2，所以权重文件是 TF 格式。加载后转换为 PyTorch。如果 TF 安装有问题，可以使用 HuggingFace 的替代方案（`ch05/02_alternative_weight_loading/`）。
