## 一、使用 LLM 的三种方式

在动手构建之前，先理解我们日常接触 LLM 的三种场景：

| 方式 | 例子 | 你能做什么 | 你不能做什么 |
|------|------|-----------|-------------|
| **公共 API 服务** | ChatGPT、Perplexity、Gemini | 提问、获取回答 | 不能定制模型行为或查看内部 |
| **本地运行开源模型** | Llama 3、Mistral（通过 Ollama/litellm） | 完全控制、可微调、数据不出本地 | 模型能力受限于开源模型 |
| **从零构建自己的模型** | **← 本书做的事** | 理解每个组件如何工作 | 不追求超越 GPT-4 |

> **本书选择方式三**：从零手写 GPT，目的是**理解底层原理**，而不是造一个比 GPT-4 更强的模型。理解了原理后，你可以更好地使用和微调任何 LLM。

## 三、阶段一：构建与预训练

### 3.1 分词（Tokenization）

**分词是把人类文本变成机器能理解的数字的第一步。**

```
原始文本: "Hello, world. This is a test."
    ↓ 分词
Token:   ['Hello', ',', ' world', '.', ' This', ...]
    ↓ 映射为整数 ID
Token ID: [15496, 11, 995, 13, 770, ...]
```

**两种主要分词方法：**

| 方法 | 原理 | 词汇量 | 使用者 |
|------|------|--------|--------|
| **简单词表** | 按空格/标点拆分，每个词一个 token | 全部唯一词（可能几万到几十万） | 早期模型 |
| **BPE**（Byte Pair Encoding） | 合并高频字符对，形成子词单元 | 固定大小（GPT-2 = 50,257） | GPT 系列、Llama |

**BPE 的核心优势**：遇到没见过的词，不会崩溃或标记为 `<|unk|>`，而是拆分为已知的子词单元：

```
未知词: "Akwirw ier"
    ↓ BPE 子词拆分
["Ak", "wir", "w", " ier"]   ← 4 个 token
```

> 视频中作者演示了 `tiktoken` 库（BPE 分词器）如何处理这种未知词——拆得越碎，说明词表越不认识这个词，效率越低，但不会失败。

### 3.2 训练数据规模

视频提到了真实 LLM 预训练的数据量级：

| 模型 | 训练数据量 | 说明 |
|------|-----------|------|
| GPT-3 | ~5000 亿 token（0.5 万亿） | 2020 年，来自公开网页 |
| GPT-4 | 未公开（估计数万亿 token） | 数据质量比数量更重要 |
| Llama 2 | 1.4 万亿 token | 2023 年，Meta 开源 |
| 本书小型 GPT | ~几百万 token | 教育目的，足够理解原理 |

> 训练数据通常只遍历 **1-2 个 epoch**（甚至不到 1 个），因为数据集太大，遍历一遍就足够模型学习了。

### 3.3 预训练的目标

```
输入:  "The cat sat on the"
目标:  "cat sat on the mat"  ← 整体向右偏移一位

模型的任务 = 给定前文 token，预测下一个 token（自回归）
```

这就是"下一个词预测"——LLM 最根本的训练目标。

### 3.4 模型架构概览

视频展示了 GPT 模型的核心组件：

```
输入 Token IDs
    ↓
Token Embedding + Positional Embedding
    ↓
Dropout
    ↓
┌─────────────────────────────┐
│  Transformer Block ×N       │
│  ├── Layer Norm             │
│  ├── Multi-Head Attention   │
│  ├── Dropout                │
│  ├── Residual Connection    │
│  ├── Layer Norm             │
│  ├── Feed Forward (GELU)    │
│  ├── Dropout                │
│  └── Residual Connection    │
└─────────────────────────────┘
    ↓  (重复 6/12/32 层)
Layer Norm
    ↓
Linear (输出层 → 词汇表大小的 logits)
    ↓
Softmax → 概率分布 → 采样下一个 token
```

**关键参数对比**（视频中提到）：

| 参数 | 小模型 | GPT-2 124M | GPT-2 355M | 7B 模型 |
|------|--------|-----------|-----------|---------|
| 嵌入维度 | 256 | 768 | 1024 | 4096 |
| 层数 | 2-6 | 12 | 24 | 32 |
| 注意力头数 | 2-4 | 12 | 16 | 32 |
| 上下文长度 | 256 | 1024 | 1024 | 4096 |

> 本书第 4 章实现 GPT-2 架构（124M 参数版本），第 5 章加载 OpenAI 发布的预训练权重。

### 3.5 训练循环要点

视频强调了预训练的几个关键实践：

| 要点 | 说明 |
|------|------|
| **Epoch 数** | 通常 1-2 个，甚至不到 1 个 |
| **Batch size** | 小模型 8-64，大模型需要分布式训练 |
| **学习率** | 通常配合 warmup + cosine decay |
| **数据采样** | 随机抽取 batch，不是严格按顺序遍历 |
| **验证** | 定期检查验证集 loss，防止过拟合 |

## 四、阶段二：监督微调（Supervised Finetuning）

预训练后的模型已经"理解"语言，但还需要学习特定任务。视频介绍了两种微调方式：

### 4.1 分类微调（Classification Finetuning）

```
预训练模型
    ↓ 替换输出层
    ↓ Linear: 768 → 2（垃圾 / 非垃圾）
    ↓ 冻结大部分层（可选）
    ↓ 在标注数据上训练
分类器模型
```

| 要点 | 说明 |
|------|------|
| **数据格式** | `(文本, 标签)` — 如 `("恭喜中奖", "spam")` |
| **输出层** | 替换原始的 50257 维输出为类别数（如 2 维） |
| **冻结策略** | 可以只训练最后一层（快但效果 ~75%），或训练所有层（慢但 ~97%） |
| **本书** | 第 6 章，垃圾邮件分类器 |

### 4.2 指令微调（Instruction Finetuning）

```
预训练模型
    ↓ 使用指令-回复对数据训练
    ↓ (如 Alpaca 格式：instruction + input + output)
    ↓
对话模型（能遵循指令回答问题）
```

| 要点 | 说明 |
|------|------|
| **数据格式** | `(指令, 输入, 期望回复)` — Alpaca 格式 |
| **数据量** | 通常 5 万到 10 万条指令-回复对 |
| **Loss 计算** | 只在回复部分计算 loss（跳过指令部分） |
| **本书** | 第 7 章，构建对话助手 |

> 视频中作者特别提到：**预训练模型已经"懂"语言了，微调只是让它学会以特定格式回答问题。** 这就是为什么微调所需的数据量远小于预训练。

## 五、阶段三：偏好调优/对齐（Preference Tuning）

**让模型的回答更符合人类偏好。**

```
指令微调后的模型
    ↓ 生成多个回答
    ↓ 人类/模型标注哪个回答更好
    ↓ 在偏好数据对上优化
对齐后的模型（回答更有用、更安全）
```

### 5.1 为什么需要偏好调优？

同一个问题可以有多种合理的回答方式：

```
问题: "买新笔记本电脑时应该关注什么？"

回答 A（技术向）: "关注 CPU 型号、RAM 容量、SSD 读写速度..."
回答 B（用户友好）: "想想它如何融入你的日常生活，如果经常出差就选轻薄本..."
```

偏好调优让模型学会哪种回答风格更受欢迎。

### 5.2 主要方法

| 方法 | 原理 | 复杂度 |
|------|------|--------|
| **RLHF** | 训练奖励模型 + PPO 强化学习 | 高（需要 3 个模型） |
| **DPO** | 直接在偏好数据对上优化，无需奖励模型 | 中（只需要 1 个模型） |

> 本书第 7 章的 bonus 部分实现了 DPO。视频没有深入细节（"这是个复杂的话题"），但指出了这是 LLM 开发的最后一个阶段。

## 六、模型评估

视频花了不少时间讲解如何评估 LLM 的表现：

### 6.1 常见评估方法

| 方法 | 说明 | 适用阶段 |
|------|------|---------|
| **Loss 曲线** | 训练/验证 loss 随 epoch 的变化 | 预训练 |
| **验证准确率** | 分类任务的准确率 | 分类微调 |
| **MMLU** | 多选题基准测试（0-100 分） | 通用能力 |
| **人类评估** | 让人判断哪个回答更好 | 指令微调/对齐 |
| **模型对比** | 用 GPT-4 等强模型当裁判 | 指令微调/对齐 |

### 6.2 视频中的评估示例

作者演示了如何用**人类评估**和**模型对比**来评估指令微调的效果：

```
输入: "买笔记本电脑要关注什么？"
模型 A 的回答 vs 模型 B 的回答
    ↓ 人类标注 / GPT-4 评分
A 胜 / B 胜 / 平局
```

> 这是偏好调优数据的来源——通过对比不同模型的输出，构建"哪个更好"的训练数据。

## 八、关键概念速查表

| 概念 | 一句话解释 |
|------|-----------|
| **Tokenization** | 把文本拆成小块（token），映射为整数 ID |
| **BPE** | GPT 用的分词算法，能处理未知词（拆成子词） |
| **Embedding** | 把整数 ID 变成稠密向量（768 维），编码"这个词是什么" |
| **Positional Embedding** | 编码"这个词在第几个位置"，GPT 用可学习的位置向量 |
| **Self-Attention** | 让每个 token 关注序列中其他所有 token |
| **Multi-Head Attention** | 多个注意力头并行，每个关注不同的关系模式 |
| **Transformer Block** | Attention + FeedForward + LayerNorm + Residual 的组合 |
| **自回归** | "给前文，预测下一个词"——LLM 的核心训练目标 |
| **Epoch** | 遍历整个训练集一次。LLM 预训练通常只做 1-2 个 epoch |
| **SFT** | 监督微调——在标注数据上训练模型做特定任务 |
| **DPO** | 直接偏好优化——用"回答 A 比 B 好"的数据训练模型 |
| **MMLU** | 多选题基准测试，给 LLM 打 0-100 分 |

## 十、常见问题

**Q: 我只有 CPU，能跑通这些内容吗？**
A: 完全可以。第 2-4 章不需要 GPU。第 5 章预训练建议用 Colab 免费 T4 GPU。

**Q: 为什么预训练只跑 1-2 个 epoch？**
A: 训练数据通常有数千亿 token，遍历一遍模型就已经学到了大量知识。跑太多 epoch 容易过拟合。实际上很多大模型连 1 个完整 epoch 都没跑完。

**Q: 微调和预训练的区别是什么？**
A: 预训练是在**大规模无标签文本**上学习通用语言表示。微调是在**小规模有标签数据**上学习特定任务。微调数据量小得多（5-10 万条 vs 数千亿 token），但足以让模型适应特定任务。

**Q: DPO 和 RLHF 哪个更好？**
A: DPO 更简单（不需要训练奖励模型），效果接近 RLHF。近年来 DPO 越来越流行。本书实现了 DPO。

**Q: 为什么要从零构建而不是直接用 HuggingFace？**
A: 理解底层原理后，你能更好地调试、优化和定制模型。HuggingFace 是"黑盒"——出了问题你知道往哪看。
