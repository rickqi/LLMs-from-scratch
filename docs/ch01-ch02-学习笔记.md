## 第 2 章：处理文本数据

### 文本→数字的完整流水线

```
原始文本
  ↓ Step 1: BPE 分词
Token IDs [910, 367, 2885, ...]
  ↓ Step 2: Embedding
Token Vectors (seq, 768)
  ↓ + Positional Embedding
Input Vectors (seq, 768)
  ↓ Step 3: 滑动窗口
(input, target) 对
  ↓ Step 4: DataLoader
(input_batch, target_batch)
  ↓
送入 GPT 训练
```

### BPE 分词 (Byte Pair Encoding)

```python
import tiktoken
tokenizer = tiktoken.get_encoding("gpt2")

text = "Every effort moves you"
ids = tokenizer.encode(text)  # [6109, 3626, 6100, 345]
decoded = tokenizer.decode(ids)  # "Every effort moves you"
```

**为什么用 BPE？**
- word-level: 词表太大 (~百万), 遇到新词 = `<UNK>`
- char-level: 序列太长, 语义弱
- BPE: ~50K 子词, 新词可分解 → 无 `<UNK>`

```
"unbelievable" → ["un", "bel", "iev", "able"]
"tokenization" → ["token", "ization"]
```

GPT-2 词表: **50,257** 个 token

### Embedding

```python
# Token Embedding: 每个 ID → 768 维向量
tok_emb = nn.Embedding(50257, 768)  # 查表: O(1)

# Positional Embedding: 每个位置 → 768 维向量
pos_emb = nn.Embedding(1024, 768)  # 可学习的位置编码

# 最终输入 = Token Emb + Position Emb
input_embeddings = tok_emb(token_ids) + pos_emb(positions)
# shape: (batch, seq_len, 768)
```

**关键洞察：nn.Embedding = 高效的 one-hot × Linear**
```python
# 两者完全等价:
Embedding(10, 5)(tensor([3]))
  ≡ Linear(10, 5, bias=False)(one_hot(3))
# Embedding 只是用索引查表, 不存储 one-hot → 更高效
```

### 滑动窗口 (Sliding Window)

```python
class GPTDatasetV1(Dataset):
    def __init__(self, txt, tokenizer, max_length, stride):
        token_ids = tokenizer.encode(txt)
        for i in range(0, len(token_ids) - max_length, stride):
            self.input_ids.append(token_ids[i : i + max_length])
            self.target_ids.append(token_ids[i+1 : i+1 + max_length])
```

| 参数 | 作用 | 典型值 |
|------|------|--------|
| max_length | 每个样本的 token 数 | 256 (教学) / 1024 (GPT-2) |
| stride | 窗口移动步数 | = max_length (无重叠) |
| batch_size | 每批样本数 | 2-32 |
| drop_last | 丢弃不完整批次 | True |

### 特殊 Token

```
GPT-2 只有一个特殊 token:
  <|endoftext|> (ID=50256)
    → 文本结束 / 文档分隔 / padding 填充

对比 BERT: [CLS] [SEP] [MASK] [PAD] [UNK] = 5 个特殊 token
```
